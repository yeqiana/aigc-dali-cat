# 《啾啾和大力》最终执行规范 V2.2｜Storyboard 导演层升级

> 生效日期：2026-07-12；适用 A-Lite / B / C 全模式。

## 1. 核心定位与公式

项目使用经典无对白剧场动画式喜剧语法：快速揭晓、极端体量反差、弹性形变、动作同步音乐、短静默、延迟反应、尾梗和荒诞物理。不得复制具体受版权保护作品的画面、角色、分镜、配乐、音效或旋律。

```text
异常已经发生 → 大力认真执行 → 啾啾直接行动 → 世界荒诞变化 → 大力延迟反应 → 尾梗
```

## 2. 正式门禁

1. 立项先通过禁重复检查并填写题材指纹。
2. 先完成 Storyboard（故事板）导演表，再生成 Storyboard 图。
3. Storyboard P0 未通过，不得规划或生成正式 Keyframe（关键帧）。
4. Keyframe P0 未通过，不得进入对应视频段。
5. 成片 P0 未通过，不得标记为已发布。

## 3. Storyboard、Keyframe、Video

- Storyboard 锁定观众在每个节点看到、理解和期待的内容，可用低精度草图或合成总览评审。
- Keyframe 锁定高风险画面的角色身份、比例、构图、道具状态和连续性，数量由风险决定。
- Video 承担实际动态；一段只承担一个清楚动作或因果，不强求单段生成完整故事。
- 高速拉镜、复杂转场和模型不稳定的连续镜头允许由独立 KF、快速缩放、运动模糊、切镜和短重音在后期实现。

## 4. A-Lite 规则

- 默认 6 张 Storyboard，合理范围 5—8 张；少于 6 张须在单集表说明原因。
- 正式 KF 数量由风险决定，低风险集可少，不再机械限定 1—2 张。
- 成本通过母版复用、低精度 SB、分段生成、剪辑、声音和失败降级控制，不得牺牲钩子、反转、延迟反应和尾梗。
- EP001 固定 6 张 SB、5 张 KF、2—3 段视频，目标时长 5.8—6.3 秒。

## 5. 角色与安全 P0

- 啾啾：白色银渐层小母猫，大头短肢四足，粉色绒绒背心与白色包边，无其他配饰；行动先于分析且不恶意欺负大力。
- 大力：奶茶色小男仓鼠，圆头短手短脚，只有一顶低饱和牛仔蓝棒球帽，无固定衣服、无长尾巴、不老鼠化；站立高度为啾啾肩高 35%—45%，含帽不超过 50%。
- 只允许安全的卡通弹性形变；不得用真实撞伤、压伤、夹伤、疼痛或恐惧制造笑点。

## 6. 画面与声音 P0

- 正式 SB、KF、视频不生成解释字幕、集名、随机文字、水印、品牌或二维码。
- 无对白、旁白、歌词；不用流行歌曲、电子舞曲、史诗预告片音乐、恐怖弦乐、持续 Pad、伤害音或完整终止和弦。
- 声音层级：关键因果音效 > 0.15—0.25 秒静默 > 动作同步配乐 > 极弱环境底噪。
- 详细声音按 07 与 08 执行；冲突时 A-Lite 采用 08 的成本流程，同时遵守 07 的共同声音边界。

## 7. 失败降级

每段动态记录尝试次数。达到上限仍不稳定时，依次减少同时运动主体、固定镜头、缩小位移，最终降级为关键帧运动、缩放、转场和声音；不得通过删除锁定剧情节点来“通过生成”。
