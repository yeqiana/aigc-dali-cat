# 《啾啾和大力》最终执行规范 V4.0｜分级生产总规范

> 状态：`main` 当前全模式总规范  
> 生效日期：2026-07-11  
> 定位：统一角色、世界观、视觉、安全和生产边界；时长、片段数、翻转数和声音复杂度由 A-Lite / B / C 模式文件决定。

---

## 1. 系列定位

```text
在普通人类居家空间里，一只好奇、先行动的白色小猫和一只认真、爱指挥的小仓鼠，
面对相对自己巨大的日常物品，通过目标错位和安全动作产生轻喜剧。
```

角色关系：

- 啾啾和大力是搭档，不是捕食关系；
- 大力负责判断、计划、指挥和努力解决；
- 啾啾负责观察、快速行动和温和偏离计划；
- 笑点来自目标错位、物理反馈和关系反应；
- 不以伤害、羞辱、疼痛、哭泣或恐惧作为笑点。

---

## 2. 模式分级

| 模式 | 适用 | 时长 | 动态 | 关键帧 | 动作链 |
|---|---|---:|---:|---:|---|
| A-Lite | 未验证题材、前 30 集 | 6—8 秒 | 1 段 | 1—2 张 | 1 个核心动作 |
| B | 数据通过后的五集故事 | 8—12 秒 | 1—2 段 | 2—3 张 | 每集 1 个节点 |
| C | 数据胜出题材精品重制 | 10—15 秒，必要时 15—20 秒 | 2—4 段 | 3—6 张 | 完整动作链 |

本文件不统一强制：

- 4—6 个生成片段；
- 两次视觉翻转；
- 完整六阶段管弦乐；
- 九格故事板；
- 15 秒成片。

这些要求必须根据模式执行。

---

## 3. 全模式共同内容公式

```text
巨大日常物品
+ 已经发生的异常
+ 大力的明确目标或判断
+ 啾啾的行动或偏离
+ 安全可读的物理结果
+ 角色关系回报
```

### A-Lite

```text
异常 → 目标/反应 → 唯一核心动作 → 轻反转或循环
```

### B

```text
本集新状态 → 本集目标 → 单一故事节点 → 结果 → 下一集接点
```

### C

```text
强异常 → 目标错位 → 动作链 → 结果/翻转 → 升级 → 情绪或视觉回报
```

## 4. 角色 P0

### 4.1 啾啾

- 白色银渐层小母猫；
- 四足小猫体态；
- 卡通化大头、短圆四肢；
- 始终穿粉色绒绒背心和白色包边；
- 不换品种、颜色、服装；
- 不增加项圈、裙子、帽子或复杂配饰；
- 不能变成真实成年猫长腿比例；
- 表演依靠瞳孔、嘴型、耳朵、身体重心和四肢。

### 4.2 大力

- 奶茶色小男仓鼠；
- 圆头、短手短脚；
- 站立高度约为啾啾肩高 35%—45%；
- 含帽最高不超过啾啾肩高 50%；
- 始终只有一顶低饱和牛仔蓝棒球帽；
- 不穿其他衣服；
- 不长尾巴、不变成老鼠、不拉成长身比例；
- 蓝帽仅作为身份标识和表情角度，不作为整集任务道具。

### 4.3 蓝帽永久边界

禁止：

- 帽子搭便车；
- 帽子作为车、船、滑板、降落伞或容器；
- 帽子丢失、被风吹走、被线勾走、被巨物带走；
- 大力整集追帽子；
- 第二顶帽子；
- 帽子无因变色、变形或消失。

允许：

- 帽檐压低；
- 帽子轻微歪斜；
- 受惊时角度变化；
- 尾帧恢复为唯一正常帽子。

## 5. 世界和场景 P0

- 主要发生在干净、生活化的居家空间；
- 日常物品相对角色呈巨大比例；
- 巨物必须实际参与动作，不只是背景；
- 允许局部人类痕迹，如家具、声音、模糊手部边缘，但不出现完整人类主角；
- 无品牌、快递单、复杂可识别文字、生成器文字、水印；
- 不出现血腥、恐怖、真实伤害；
- 巨物动作安全，可滑动、弹起、晃动、滚动、折叠、展开、摆动和遮挡；
- 不允许尖锐结构刺向角色、重物压住角色、角色被夹住或无法脱困。

---

## 6. 第一帧规则

第一帧不是场景介绍，必须已经发生事件。

至少包含以下三项：

- 巨物或异常结构；
- 明确运动方向或形变趋势；
- 至少一个角色身体反应；
- 目标或阻碍；
- 反常结果；
- 高识别度色彩或明暗分离。

禁止：

- 两只角色静止站着看；
- 巨物稳定摆放；
- 缓慢推镜介绍房间；
- 先出现标题再发生事件；
- 只靠字幕、旁白或发布文案解释。

## 7. 动作和表演

### 7.1 单动态基本结构

```text
预备 → 主动作 → 结果
```

### 7.2 动作可读性

必须写明：

- 起点；
- 方向；
- 速度变化；
- 物体反馈；
- 角色结果姿势；
- 尾帧状态。

### 7.3 主要反应

每个主要反应至少改变两项：

- 眼睛或瞳孔；
- 嘴型；
- 耳朵方向；
- 身体重心；
- 四肢姿势；
- 大力帽子角度。

### 7.4 动画形变

允许安全、短暂的挤压和拉伸，但：

- 不改变角色总体身份；
- 不增加肢体；
- 不产生畸形爪子；
- 尾帧恢复锁定比例。

## 8. 视觉风格

统一底座：

```text
高品质 3D 动画电影角色，卡通化头身比例，柔软块状绒毛，
清晰眉眼和夸张但可爱的嘴型，强轮廓表演，立体电影光影，
生活化材质，非真实宠物摄影，非逐根超写实毛发，非低龄平涂 Q 版。
```

画面要求：

- 啾啾粉色背心和大力蓝帽清楚可见；
- 巨物、角色和背景分离；
- 角色脸部在手机缩略图尺寸可读；
- 背景不抢主体；
- 运动方向一侧保留空间；
- 一张关键帧只有一个主要动作或状态重点。

## 10. 禁重复

立项前必须查阅：

```text
docs/strategy/已发布内容与禁重复清单.md
```

题材指纹固定为：

```text
道具 + 核心运动 + 大力目标 + 啾啾偏离 + 最终结果
```

以下为永久禁区：

- 帽子搭便车及所有帽子被带走、追帽变体；
- 巨大西瓜或相似巨大圆形水果滚动追逐；
- 快递盒、纸箱被啾啾当家、卧室或入住空间；
- 容器滑入后啾啾钻入并从孔洞探头，若整体剧情与已发布内容高度相似；
- 蓝帽作为交通工具、任务道具或失踪物。

## 11. 模式选择门槛

### 11.1 A-Lite

满足：

- 未验证；
- 一个动作；
- 6—8 秒能讲清；
- 不需要多场景、复杂救援和新角色；
- 当前以免费额度为主。

### 11.2 B

必须：

- 对应动作结构已有 A-Lite 数据；
- 满足至少两项升级条件；
- 4 次以内可稳定生成，或有明确降级方案；
- 五集故事每集指纹不同；
- 不重新演一遍 A-Lite 测试。

### 11.3 C

必须：

- 至少两条免费版本验证；
- 角色一致性可控；
- 有明确视觉高潮或情绪回报；
- 付费后能明显提升；
- 有预算和止损；
- 不是只增加时长。

---

## 12. 全模式共同一票否决

- 未做禁重复检查；
- 使用错误模式模板；
- 角色身份、服装、比例或唯一帽子漂移；
- 大力长尾巴、穿衣服、变成老鼠或接近啾啾体型；
- 巨物尺寸在相邻镜头无因跳变；
- 角色被压伤、夹伤、撞伤或困住；
- 完整人类、品牌、随机文字、水印；
- 无声观看完全无法理解；
- 依赖字幕解释主要动作；
- 为凑时长保留无信息停顿；
- 超过模式尝试上限仍继续消耗；
- 未完成批次复盘就批量进入下一阶段。
