# 《啾啾和大力》A-Lite 低成本生产规范 V1.2｜Storyboard 导演层升级

> 文件名为兼容现有引用继续保留 `V1.0`；正文由 V1.1 最小升级为 V1.2。
> 生效日期：2026-07-12；适用未验证题材及路线图 A-Lite 池。

## 1. 低成本目标

在不降低角色身份、比例、安全、禁重复与关键喜剧节点 P0 的前提下，减少高精度正式图、动态尝试、复杂声音和无数据支持的精品投入。Storyboard（故事板）导演层用于提前发现问题，不等于生成 6 张高成本正式图。

## 2. 标准配置

| 项目 | A-Lite 规则 |
|---|---|
| Storyboard | 默认 6 张，合理范围 5—8 张；可先用低精度草图或六格合成总览 |
| 正式 Keyframe（关键帧） | 由风险决定；低风险集可少，高风险因果、运镜前后和首尾状态应独立锁定 |
| 视频 | 按清楚动作或因果拆分；不要求一段完成全部故事 |
| 场景 | 优先 1 个，通过母版复用控制成本 |
| 单段动态尝试 | 最多 4 次，达到上限必须降级 |
| 声音 | 简化时间轴，同时遵守 07 的共同声音边界 |

EP001 固定 6 张 SB、5 张 KF、2—3 段视频，不能在生产启动阶段缩减。

## 3. 流程

```text
禁重复 → 导演表 → 低精度 Storyboard → Storyboard P0
→ 高风险 Keyframe → Keyframe P0 → 分段视频 → 剪辑与声音 → 成片 P0
```

Storyboard P0 未通过不得进入正式 KF。独立 SB 图用于逐张生产与验收；六格总览只用于导演预览，不能替代 6 张独立交付。

## 4. 不得牺牲的结构

A-Lite 成本控制不得删除钩子、第一次反转、延迟反应或尾梗。默认六段为 Hook、Reveal、Escalation、First Twist、Delayed Reaction、Ending Gag；少于 6 张须在单集生产表说明剧情为何仍完整。

## 5. Keyframe 与视频拆分

- 只有角色身份易漂移、状态跨越大、复杂接触、首尾帧或高风险运镜前后才优先制作正式 KF。
- 相邻 SB 若机位、站位和道具状态基本不变，可共用一张 KF。
- 每个视频段只承担一个清楚动作或因果；双角色同时高速运动不稳定时，优先让道具运动，另一角色只做反应。
- 快速拉镜和复杂转场优先使用独立 KF、切镜、快速缩放、运动模糊与声音重音后期实现。

## 6. 简化声音

声音层级：

```text
关键因果音效 > 0.15—0.25 秒静默 > 动作同步配乐 > 极弱环境底噪
```

最小配置包括异常/巨物音、角色动作音、结果音、短静默、可复用无对白轻喜剧动作同步配乐和未解决尾音。禁止对白、旁白、歌词、流行歌曲、电子舞曲、史诗预告片音乐、恐怖弦乐、持续 Pad、真实伤害音、疼痛叫声和完整终止和弦。07 提供共同声音边界，08 决定 A-Lite 的简化实施与成本。

## 7. 四次止损与降级

1. 第 1 次只修一个最大问题；
2. 第 2 次减少一个同时运动主体；
3. 第 3 次固定镜头、降低速度、缩小位移；
4. 第 4 次为最后一次，失败后停止该动态生成。

失败后按顺序降级为静态首尾帧、关键帧运动、快速缩放、转场、轻震动和声音补足。降级只能改变实现方式，不得删除已锁定剧情节点。

## 8. P0

- [ ] 已通过禁重复并填写题材指纹；
- [ ] Storyboard 数量、职责和状态真实；
- [ ] 第一秒钩子/揭晓成立，3 秒内出现第一次反转；
- [ ] 延迟反应和尾梗可独立读懂；
- [ ] 角色身份、服装、比例、空间方向与巨物状态连续；
- [ ] 无真实伤害、品牌、文字、水印、完整人类或额外角色；
- [ ] 关键因果音、短静默、动作同步和未解决尾音存在；
- [ ] 尝试次数与降级记录完整；
- [ ] 未因低成本删除关键喜剧节点。

## 9. 升级门槛

是否进入 B/C 仍依据发布后的真实数据、生成稳定性与预算批准；不得把播放预估写成保证，也不得记录未确认的模型额度或成本。
