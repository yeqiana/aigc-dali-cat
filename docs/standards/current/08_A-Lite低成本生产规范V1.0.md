# 《啾啾和大力》A-Lite 低成本生产规范 V1.1

> 文件名为兼容现有引用继续保留 `V1.0`，正文版本升级为 V1.1。  
> 生效日期：2026-07-11  
> 适用：路线图第 1—30 集和任何未验证题材。

---

## 1. 目标

在不降低角色身份、比例、安全、无品牌和禁重复 P0 的前提下，降低：

- 动态段数量；
- 关键帧数量；
- 声音复杂度；
- 单镜头尝试次数；
- 无数据支持的精品投入。

---

## 2. 固定规格

| 项目 | A-Lite |
|---|---|
| 成片 | 6—8 秒 |
| 动态 | 1 段 |
| 动态时长 | 3—5 秒 |
| 关键帧 | 1—2 张 |
| 场景 | 1 个 |
| 核心动作 | 1 个 |
| 结果 | 1 个轻反转或循环即可 |
| 单动态尝试 | 最多 4 次 |
| 声音 | 简化时间轴 |
| 配角 | 不增加 |
| 生产目的 | 测动作、角色关系、稳定性和数据 |

不要求：

- 4—6 个动态片段；
- 九格正式故事板；
- 两次翻转；
- 完整六阶段管弦乐；
- 先补齐全部母版；
- 一次做完 5 条成片。

---

## 3. 立项门禁

必须：

1. 查阅 `docs/strategy/已发布内容与禁重复清单.md`；
2. 填写题材指纹；
3. 选择 `02A_A-Lite单集生产表.md`；
4. 确认一个核心动作；
5. 写明 4 次止损和降级方案。

永久禁止：

- 帽子搭便车和追帽；
- 巨大西瓜或同类圆形水果追逐；
- 快递盒/纸箱入住；
- 蓝帽成为道具、交通工具或失踪物；
- 新增小兔、乌鸦等配角；
- 复杂缠绕、危险碰撞和真实救援。

---

## 4. 时间结构

| 时间 | 功能 |
|---:|---|
| 0.0—0.8 秒 | 巨物已经运动或变形 |
| 0.8—2.0 秒 | 大力目标或啾啾行动可见 |
| 2.0—5.5 秒 | 唯一核心动作 |
| 5.5—7.5 秒 | 结果、轻反转或循环 |

第一帧至少包含三项：

- 巨物/异常结构；
- 运动方向或形变趋势；
- 角色身体反应；
- 主辅色分离；
- 不看字幕也会产生问题。

---

## 5. 关键帧

- 默认 1 张强首帧；
- 需要锁定结果时增加 1 张结果帧；
- 不生成九格作为必需交付；
- 每张图只有一个画面重点；
- 预留运动方向空间；
- 啾啾背心和大力唯一蓝帽可见；
- 大力保持啾啾肩高 35%—45%；
- 不出现标题、对白、编号、品牌、随机文字和多格排版。

---

## 6. 动态

只允许：

```text
预备 → 主动作 → 结果
```

提示词必须写：

- 起点；
- 方向；
- 速度变化；
- 物体反馈；
- 结果姿势；
- 尾帧。

双角色同时高速运动不稳定时：

1. 保留一只角色运动；
2. 另一只角色只做清楚反应；
3. 优先让道具运动；
4. 降低速度和位移。

---

## 7. 简化声音

最小配置：

- 一条可复用的无对白轻喜剧 BGM；
- 一个巨物声音；
- 一个角色动作声音；
- 一个结果声音；
- 0.15—0.25 秒短静默；
- 极短未终止尾音。

模板：

```text
0.0—0.8 秒：巨物事件音 + BGM 立即进入
0.8—2.0 秒：轻快蓄力或观察音型
2.0—5.5 秒：核心动作因果音，BGM 随速度略变化
结果前：0.15—0.25 秒静默
5.5—7.5 秒：轻巧结果音 + 极短循环尾音
```

---

## 8. 四次止损

1. 第 1 次：只修一个最大问题；
2. 第 2 次：减少一个同时运动的角色；
3. 第 3 次：降低速度、缩小位移、固定镜头；
4. 第 4 次：最后一次，失败后停止。

失败后：

- 静态首尾帧；
- 剪辑推镜、速度变化、轻震动；
- 音效补中间过程；
- 更换下一题材。

不得因“已经试了很多次”继续消耗免费或付费额度。

## 9. P0

- [ ] 去重通过；
- [ ] 第一帧已发生事件；
- [ ] 1 秒内识别巨物和方向；
- [ ] 2 秒内看懂目标或行动；
- [ ] 全片一个核心动作；
- [ ] 无声可懂；
- [ ] 角色身份、服装、比例连续；
- [ ] 大力唯一帽子、无衣服、无尾巴；
- [ ] 巨物 > 啾啾 > 大力；
- [ ] 同一透视地面；
- [ ] 无真实伤害和危险感；
- [ ] 无品牌、文字、水印、完整人类和额外角色；
- [ ] 有最小声音因果和短静默；
- [ ] 无凑时长空镜；
- [ ] 尝试次数已记录；
- [ ] 正式导出达标。

---

## 10. 升级门槛

满足任意两项，可进入 B 候选：

- 完播率高于近 10 条平均；
- 平均播放时长高于近 10 条平均；
- 重播率明显提升；
- 评论主动出现角色名；
- 主页访问或涨粉高于平均；
- 4 次以内稳定生成；
- 同类动作结构连续两条不差。

进入 C 还需：

- 至少 2 条免费版本验证；
- 角色一致性可控；
- 有明确高潮或情绪回报；
- 付费重制能明显提升，而非仅增加时长。
