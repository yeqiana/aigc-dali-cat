# 《啾啾和大力》成片评审与发布验收规范 V1.0

> 文档性质：成片阶段补充规范  
> 适用范围：《啾啾和大力》全部短视频单集  
> 建议路径：`docs/standards/current/09_成片评审与发布验收规范.md`  
> 上游规范：`01_最终执行规范.md`、`02_单集生产表.md`、`05_尺寸与比例规范.md`  
> 核心原则：本文件只补充“成片评审、发布验收和数据复盘”，不得降低现有角色身份、比例、安全边界和 P0 标准。

---

## 1. 文档目标

本规范用于解决以下问题：

1. 关键帧单独看没有问题，但合成视频后节奏拖沓；
2. 角色外形稳定，但角色目标和性格没有通过动作表现出来；
3. 有一个视觉笑点，但缺少完整的目标、冲突、翻转和循环；
4. 画面好看，但无声观看时无法理解剧情；
5. 成片含水印、低分辨率或错误帧率，无法直接正式发布；
6. 单集复盘停留在主观感受，没有形成可回写的结构化结论。

成片验收必须同时检查：

```text
故事是否成立
+
节奏是否有效
+
角色是否在表演
+
视觉是否连续
+
声音是否推动剧情
+
文件是否达到发布标准
```

只满足其中部分，不得判定为正式发布通过。

---

## 2. 成片评审的三层结论

每条成片评审只允许使用以下三种结论。

### 2.1 正式发布通过

满足全部 P0 项目，可以进入正式发布。

### 2.2 修改后发布

没有角色身份、安全或文件硬伤，但存在可通过剪辑、音效、字幕或局部替换修复的问题。

典型情况：

- 中段停顿略长；
- 个别动作音不够清楚；
- 某一反应停留时间过长；
- 首尾循环接点不够顺滑；
- 字幕出现时间可以优化。

### 2.3 不通过，需要重制

任意 P0 失败，或者问题无法仅靠后期剪辑修复。

典型情况：

- 前 3 秒目标和冲突无法看懂；
- 角色比例明显错误；
- 角色身份、服装或配饰漂移；
- 只有动作，没有完整因果；
- 缺少第二次翻转或循环；
- 关键角色全程没有有效表演；
- 含平台或生成工具水印；
- 分辨率和导出参数不达标；
- 需要依赖大量字幕才能解释剧情。

## 3. 成片故事结构硬规则

### 3.1 第一帧必须已经发生事件

第一帧不是环境介绍，也不是两只角色站着观察。

第一帧至少同时出现以下三项：

- 巨物或异常物体；
- 明确运动方向；
- 至少一个角色的身体反应；
- 目标或阻碍；
- 反常结果；
- 高识别度色彩对比。

禁止把以下内容作为唯一开头：

- 两只角色静止站立；
- 角色缓慢走入画面；
- 巨物稳定摆放；
- 缓慢推镜展示房间；
- 依赖标题说明发生了什么。

### 3.2 前 3 秒必须完成三个信息

```text
异常事件
→
角色目标
→
角色偏离或冲突
```

建议时间：

| 时间 | 必须表达 |
|---:|---|
| 0–0.6 秒 | 什么东西不正常 |
| 0.6–1.8 秒 | 大力想完成什么 |
| 1.8–3.0 秒 | 啾啾如何偏离大力的目标 |

如果观众看完前三秒仍不知道“大力想做什么”，则目标建立失败。

如果啾啾只是随机玩耍，而不是偏离大力的明确计划，则角色冲突失败。

### 3.3 每条视频必须具有完整因果

最低因果链：

```text
事件出现
→
大力建立计划
→
啾啾误解或先行动
→
动作爆发
→
第一次翻转
→
第二次翻转或循环接点
```

不允许只表现：

```text
角色玩物体
→
物体突然飞走
→
结束
```

这种结构只有物理意外，没有角色目标错位，不能充分体现系列关系。

### 3.4 第一次翻转和第二次翻转必须区分

#### 第一次翻转

打破观众对动作结果的直接预期。

示例：

- 帽子被毛线勾走；
- 箱盖突然弹开；
- 拖鞋开始滑动；
- 雨伞突然翻面。

#### 第二次翻转

在第一次翻转后产生新的角色关系、视觉结果或喜剧后果。

示例：

- 帽子落到啾啾头上；
- 大力追帽子反而被带着跑；
- 巨物停下后又重新启动；
- 啾啾以为成功，下一秒物体再次偏离；
- 大力刚松口气，新的异常又出现。

第二次翻转不得只是同一动作继续移动得更远。

### 3.5 结尾优先形成循环

最后 0.3–0.8 秒应尽量提供可以接回第一帧的动作接点。

常用循环方式：

- 啾啾再次抬爪，接回开头的扑击；
- 巨物滚出画面，又从另一侧进入；
- 大力重新站回指挥位置；
- 绳子再次绷紧；
- 角色回头看向即将发生的新异常。

循环不是简单复制首帧，而是让结尾动作自然导向下一轮开头。

## 4. 节奏与信息密度规范

### 4.1 每 0.8–1.5 秒必须发生有效状态变化

有效变化包括：

- 角色姿势变化；
- 角色表情变化；
- 巨物状态变化；
- 角色关系变化；
- 观众已知信息变化；
- 声音强弱变化；
- 动作方向变化；
- 构图中的主体关系变化。

以下不算有效变化：

- 只有镜头轻微推进；
- 角色继续保持同一个趴姿；
- 同一动作持续时间变长；
- 只有景别变化，故事状态不变；
- 背景音乐持续播放，但没有动作反馈。

### 4.2 单个动作片段必须包含三状态

```text
预备
→
爆发
→
结果
```

示例：

```text
大力按住帽子后仰蓄力
→
毛线突然绷紧，大力双脚打滑
→
帽子被带走，大力全身僵住
```

如果片段只有“持续拉扯”，没有明确预备和结果，则动作信息不完整。

### 4.3 反应停留时间控制

角色反应只保留到观众读懂。

建议：

- 普通反应：0.3–0.6 秒；
- 翻转后的主要表情：0.4–0.8 秒；
- 除非用于制造悬念，不保留超过 1 秒的无动作停顿。

### 4.4 单一镜头的额外要求

允许整条视频使用单一镜头，但必须满足以下条件：

- 角色或巨物持续改变画面关系；
- 表情在手机尺寸下仍可读；
- 每 0.8–1.5 秒有状态变化；
- 关键动作不会因远景而看不清；
- 镜头内完成完整目标、冲突和翻转。

如果单一大全景导致角色脸部过小、目标看不懂，应拆为大全景、中景或反应近景。

## 5. 角色表演规范

### 5.1 大力必须主动参与剧情

大力不能只是站在旁边、拉着道具或充当比例参照。

大力至少应完成一项主动行为：

- 指挥；
- 解释；
- 阻止；
- 推、拉、顶、搬；
- 保护帽子或道具；
- 发现异常；
- 追赶；
- 无奈收拾结果。

观众应能通过动作判断“大力想完成什么”。

### 5.2 啾啾必须表现“先行动和温和偏离”

啾啾的行为应是对大力计划的误解、偏离或过度执行，而不是无原因随机行动。

推荐结构：

```text
大力给出明确方向
→
啾啾锁定其中一个动作目标
→
啾啾快速执行
→
执行方向与大力真实目的发生错位
```

### 5.3 每个主要反应至少改变两项表演元素

可改变元素：

- 眼睛或瞳孔；
- 嘴型；
- 耳朵方向；
- 身体重心；
- 四肢姿势；
- 尾巴方向；
- 大力帽子角度。

仅轻微睁眼或轻微张嘴，不算完整反应。

### 5.4 角色表演参考

#### 啾啾

| 状态 | 表演要求 |
|---|---|
| 锁定目标 | 身体压低、耳朵前倾、瞳孔集中 |
| 行动爆发 | 前肢伸展、身体前冲、嘴角兴奋 |
| 突然受惊 | 耳朵竖起、瞳孔放大、身体后仰 |
| 无辜发愣 | 圆眼、嘴巴微张、身体僵住 |
| 小得意 | 抬下巴、回看大力、尾巴轻抬 |

#### 大力

| 状态 | 表演要求 |
|---|---|
| 认真指挥 | 帽檐压低、双爪指向目标 |
| 拼命用力 | 眼睛挤紧、嘴巴咬紧、短脚打滑 |
| 发现翻车 | 瞳孔放大、帽子歪斜、身体僵住 |
| 无奈崩溃 | 肩膀下垂、嘴角下拉、双爪摊开 |
| 快速追赶 | 身体前倾、短腿高频移动 |

## 6. 角色连续性与比例验收

### 6.1 啾啾

必须保持：

- 白色银渐层小母猫；
- 卡通化大头短肢；
- 四足小猫体态；
- 粉色绒绒背心；
- 白色包边；
- 相同脸型和毛色。

禁止：

- 换品种；
- 换衣服；
- 增加配饰；
- 变成真实成年猫比例；
- 长时间以人形双足站立；
- 相邻镜头体型明显跳变。

短暂后脚站立可作为动作爆发，但必须快速回到四足体态。

### 6.2 大力

必须保持：

- 奶茶色小男仓鼠；
- 圆头、短手短脚；
- 只戴一顶低饱和牛仔蓝棒球帽；
- 不穿其他衣服；
- 不出现长尾巴；
- 不变成老鼠。

### 6.3 同框比例

硬规则：

```text
巨大物品 > 啾啾 > 大力
```

大力相对啾啾：

- 推荐为啾啾肩高的 35%–45%；
- 含帽最高不超过啾啾肩高的 50%；
- 必须明显小于啾啾；
- 两者同地面时必须位于同一透视平面；
- 不得通过把大力放远来伪造体型差。

### 6.4 动态比例检查

动画允许短暂挤压和拉伸，但：

- 动作尾帧必须恢复锁定比例；
- 帽子可歪斜或被带走，但始终只有一顶；
- 巨物在相邻镜头中不得明显变大或变小；
- 不得因镜头切换让大力突然接近啾啾体型。

## 7. 巨物和场景验收

### 7.1 巨物不能只是背景

巨物至少应拥有一个可参与剧情的结构：

- 可滚动；
- 可滑动；
- 可弹起；
- 可晃动；
- 可折叠；
- 可拖拽；
- 可产生遮挡；
- 可带动角色或配饰。

### 7.2 巨物动作必须产生角色反馈

巨物发生变化后，至少一名角色必须出现明显反应。

不允许：

```text
巨物移动
+
角色继续保持原姿势
```

推荐：

```text
巨物绷紧
→
大力脚底打滑
→
啾啾耳朵转向并继续追击
```

### 7.3 色彩与明暗分离

必须确保：

- 巨物与背景色相或明度有明显区分；
- 啾啾粉色背心可见；
- 大力蓝色帽子可见；
- 角色轮廓在手机缩略图中可辨认；
- 背景不抢夺主体。

## 8. 无声观看验收

每条视频发布前必须完整静音观看一次。

静音状态下必须能回答：

1. 什么东西异常？
2. 大力想做什么？
3. 啾啾为什么开始行动？
4. 两者的目标在哪里发生错位？
5. 第一次翻转是什么？
6. 第二次翻转或循环是什么？

如果其中任意两项无法回答，说明画面因果不足。

字幕只能：

- 强化冲突；
- 补充一个笑点；
- 提供一句角色态度。

字幕不得承担解释完整剧情的任务。

---

## 9. 声音验收规范

每条视频至少包含：

- 一个明确的巨物声音；
- 一个角色动作声音；
- 一个短静默；
- 一个翻转音效；
- 一个结尾反应音。

推荐声音结构：

```text
强事件音
→
短静默
→
快动作音
→
停顿
→
翻转音效
→
短反应音
```

### 9.1 禁止声音问题

- 背景音乐从头到尾同一强度；
- 所有动作共用同一个软弱音效；
- 翻转时没有声音强调；
- 结尾没有反应音；
- 音效比画面动作提前或延后明显；
- 峰值爆音；
- 使用带明显版权风险或平台水印的素材。

### 9.2 声音不得替代视觉因果

即使音效能提示“帽子飞走”，画面仍必须清楚表现帽子如何被勾住、被带动和离开。

## 10. 水印、文字和品牌 P0 规则

正式发布文件禁止出现：

- 生成平台水印；
- 剪辑软件水印；
- 模型名称；
- 品牌 Logo；
- 快递单；
- 复杂可识别文字；
- 非剧情需要的账号角标；
- 测试时间码；
- 调试文字。

用于内部评审的版本可以含时间码，但正式发布前必须导出无水印版本。

任意非项目自有水印存在，P0 直接不通过。

---

## 11. 导出文件规范

正式交付建议：

| 项目 | 标准 |
|---|---|
| 画幅 | 9:16 |
| 分辨率 | 1080 × 1920 |
| 视频编码 | H.264 High Profile |
| 帧率 | 恒定 25 或 30 fps |
| 视频码率 | 8–12 Mbps |
| 音频编码 | AAC |
| 音频采样率 | 48 kHz |
| 音频声道 | 立体声 |
| 音频码率 | 192 kbps |
| 峰值 | 不高于 -1 dBFS |
| 封装 | MP4 |

以下情况不得作为正式发布母文件：

- 720 × 1280 测试导出；
- 24 fps 与后续工程帧率不一致；
- 低码率导致毛发、地毯或毛线出现明显色块；
- 32 kHz 音频；
- 可变帧率造成剪辑或平台转码异常；
- 带水印文件；
- 重复压缩后的二次导出文件。

## 12. 成片评审标准流程

### 12.1 文件基础检查

记录：

```text
文件名：
时长：
分辨率：
帧率：
视频码率：
音频采样率：
是否有水印：
是否有黑边：
是否有首尾空帧：
```

### 12.2 第一次观看：正常带声音

只判断：

- 第一印象是否能看懂；
- 笑点出现在哪里；
- 是否有明显拖沓；
- 结尾是否还想继续看。

不得在第一次观看时逐帧找错，以免忽略普通观众的真实感受。

### 12.3 第二次观看：静音

检查故事目标、角色冲突和翻转能否无声理解。

### 12.4 第三次观看：逐段暂停

建议按 0.25 秒或 0.5 秒抽帧，检查：

- 动作是否连续；
- 是否存在长时间同状态；
- 角色比例是否变化；
- 帽子、服装和肢体是否漂移；
- 巨物比例是否跳变；
- 第一帧和尾帧是否形成有效接点。

### 12.5 第四次观看：手机缩略图

缩小画面后检查：

- 第一眼是否看见巨物；
- 角色轮廓是否分离；
- 啾啾背心是否可见；
- 大力帽子是否可见；
- 表情是否仍然可读；
- 画面是否过暗或背景过乱。

## 13. P0 正式发布验收表

任意一项失败，整条视频不得判定为正式发布通过。

### 13.1 故事与节奏

- [ ] 第一帧已经发生异常事件；
- [ ] 0.6 秒内能识别巨物和角色反应；
- [ ] 前 3 秒能看懂大力目标；
- [ ] 前 3 秒能看懂啾啾偏离；
- [ ] 无声观看可以理解主要剧情；
- [ ] 每 0.8–1.5 秒至少发生一次有效状态变化；
- [ ] 每个动作片段具有预备、爆发和结果；
- [ ] 第一次翻转清楚；
- [ ] 存在第二次翻转或循环接点；
- [ ] 成片没有为凑时长保留的无信息停顿。

### 13.2 角色与比例

- [ ] 啾啾身份、脸型、毛色和背心连续；
- [ ] 大力身份、脸型和奶茶色毛色连续；
- [ ] 大力始终只有一顶蓝帽子；
- [ ] 大力没有衣服、长尾巴或老鼠化；
- [ ] 大力约为啾啾肩高 35%–45%；
- [ ] 两个角色处于同一透视地面；
- [ ] 主要反应至少改变两项表演元素；
- [ ] 大力有主动目标和动作；
- [ ] 啾啾的行动体现对目标的偏离；
- [ ] 无额外肢体、畸形爪子和体型跳变。

### 13.3 巨物与安全

- [ ] 巨物为画面最大主体；
- [ ] 巨物与背景有色彩或明暗分离；
- [ ] 巨物实际参与动作；
- [ ] 巨物状态在相邻镜头中连续；
- [ ] 动作没有真实撞伤、压伤、夹伤或危险感；
- [ ] 角色没有以疼痛、哭泣或恐惧作为笑点。

### 13.4 声音与文件

- [ ] 至少有巨物音、角色动作音、短静默、翻转音和结尾反应音；
- [ ] 声音与动作同步；
- [ ] 背景音乐没有填满全部节奏点；
- [ ] 画面没有水印、品牌或测试文字；
- [ ] 分辨率达到 1080 × 1920；
- [ ] 帧率为恒定 25 或 30 fps；
- [ ] 视频码率达到 8–12 Mbps；
- [ ] 音频为 AAC 48 kHz；
- [ ] 音频峰值不高于 -1 dBFS。

## 14. P1 优化项

P1 不决定是否可以发布，但影响账号长期质量和复看率。

- [ ] 第一帧缩略图有清晰视觉焦点；
- [ ] 巨物运动方向与画面留白一致；
- [ ] 景别变化服务于新信息；
- [ ] 角色脸部在手机尺寸下可读；
- [ ] 首次翻转前有明确蓄力；
- [ ] 第二次翻转比第一次更具角色关系；
- [ ] 结尾循环没有明显硬切；
- [ ] 动作音具有材质感；
- [ ] 片名和封面不会提前剧透全部结果；
- [ ] 画面有一个可被用户记住的视觉姿势；
- [ ] 视频可在关闭字幕时成立；
- [ ] 视频可被压缩到更短时长而不损失剧情。

## 15. 成片评分表

评分用于比较版本，不用于替代 P0。

P0 失败时，即使总分较高，也不得正式发布。

| 维度 | 分值 | 评分重点 |
|---|---:|---|
| 前 3 秒钩子 | 20 | 异常、目标、冲突是否快速成立 |

- 90–100：正式发布优先版本；

- 60–69：需要明显重剪或补镜头；
- 60 以下：建议重制。

## 17. 从单集问题回写规范的原则

不是每次成片问题都立即修改总规范。

只有满足以下任一条件，才建议回写：

1. 至少两条正式视频重复出现同一问题；
2. 问题属于角色身份、比例或安全硬伤；
3. 问题导致明显的前 2 秒流失；
4. 问题在当前单集生产表中没有检查入口；
5. 问题可以转化成明确、可验收的规则。

不应回写：

- 某一集独有的剧情失败；
- 某次生成模型偶发畸形；
- 单个平台的临时热点；
- 仅凭个人喜好提出的镜头风格变化；
- 无法量化或验收的模糊要求。

## 19. 通用重制判断

出现以下情况，不要继续通过加速、字幕或音效勉强修复，应直接重制相关片段：

- 大力比例错误；
- 角色身份或服装漂移；
- 帽子无因消失或复制；
- 大力没有明确目标；
- 啾啾行动与大力目标没有关系；
- 第一次翻转前没有动作因果；
- 全片只有一次结果；
- 关键动作在远景中看不清；
- 画面带无法去除的水印；
- 巨物没有参与动作；
- 角色连续 1.5 秒以上保持相同姿势且无新信息。

以下问题通常可以后期修复：

- 单个反应停留过长；
- 开头多出少量空帧；
- 某个音效不够清楚；
- 翻转前静默时间略长；
- 结尾多停留 0.3–0.5 秒；
- 字幕重复画面信息；
- 背景音乐持续过满。

## 20. 最终发布口令

正式发布前，验收负责人必须能够完整确认：

```text
第一帧有事，
前三秒有目标和冲突，
中间没有无效停顿，
两个角色都在表演，
比例和身份没有漂移，
巨物实际参与动作，
至少有两次视觉结果，
结尾能够翻转或循环，
静音也能看懂，
声音推动节奏，
文件无水印且参数达标。
```

任意一句无法确认，成片不得标记为“正式发布通过”。
