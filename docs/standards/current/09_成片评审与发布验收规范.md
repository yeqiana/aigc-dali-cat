# 《啾啾和大力》成片评审与发布验收规范 V1.2｜Storyboard 对齐版

> 生效日期：2026-07-12；适用全部单集。结论只允许“正式发布通过 / 修改后发布 / 不通过”。

## 1. 评审流程

文件检查 → 带声音观看 → 静音观看 → 对照已通过 Storyboard 逐节点观看 → 逐帧连续性 → 手机缩略图 → 技术参数 → 结论与复盘。

## 2. Storyboard 对齐 P0

- [ ] 第一秒兑现 SB01 的钩子并完成或启动 SB02 揭晓；
- [ ] 3 秒内出现第一次清楚反转；
- [ ] 延迟反应无需字幕即可读懂；
- [ ] 尾梗独立成立，不只是重复第一次反转；
- [ ] 不存在无效铺垫、空镜或为凑时长的等待；
- [ ] 不依赖字幕、集名、对白或旁白解释；
- [ ] 未出现角色身份、服装、比例或帽子漂移；
- [ ] 未出现动作因果丢失或道具状态跳变；
- [ ] 声音先后与画面因果一致；
- [ ] 反转前保留 0.15—0.25 秒短静默；
- [ ] 结尾没有完整终止和弦；
- [ ] 未为生成方便删除钩子、反转、延迟反应或尾梗。

任一项失败，不得“正式发布通过”。关键节点缺失、因果无法读懂或需要大量字幕解释时必须重制。

## 3. 角色、比例与安全 P0

- [ ] 啾啾为同一白色银渐层小母猫，粉色绒绒背心与白色包边连续，无额外配饰；
- [ ] 大力为同一奶茶色小男仓鼠，只有一顶低饱和牛仔蓝帽，无固定衣服、长尾巴或老鼠化；
- [ ] 大力站立高度为啾啾肩高 35%—45%，含帽不超过 50%；
- [ ] 两者处于同一透视地面，无额外肢体与穿模；
- [ ] 巨物尺寸、材质、位置和运动方向在相邻镜头连续；
- [ ] 只有安全卡通形变，无真实撞伤、压伤、夹伤、疼痛或恐惧。

## 4. 画面、声音与文件 P0

- [ ] 无随机文字、解释字幕、集名、水印、品牌、二维码、完整人类或额外动物；
- [ ] 无对白、旁白、歌词、流行歌曲、电子舞曲、史诗预告片音乐、恐怖弦乐、持续 Pad 或伤害音；
- [ ] 关键因果音优先于配乐，动作同步，环境底噪极弱；
- [ ] 9:16，1080×1920，H.264 High Profile，恒定 25 或 30 fps，视频码率 8—12 Mbps；
- [ ] AAC、48 kHz、立体声、192 kbps，真峰值不高于 -1 dBFS。

## 5. 模式补充

- A-Lite：不按 KF 数量判定成本；检查导演节点完整、单段动作清楚、尝试次数或降级记录真实。
- B：检查本集新状态、与上一集连续和下一集接点，不重演 A-Lite 首次异常。
- C：检查精品化提升、高潮、最终回报、完整声音与预算门禁。

## 6. 修改与重制边界

可后期修复：轻微停留时长、音效清晰度、循环硬切、轻微构图裁切。必须重制：角色/比例错误、帽子复制或成为道具、核心因果缺失、真实危险、题材高相似、水印无法去除、关键节点被删、无声看不懂或需字幕解释。

## 7. 发布后复盘

记录发布时间、时长、各段尝试次数、实际成本、2 秒跳出、完播率、平均播放时长、重播率、互动率、主页访问、新增粉丝、角色名评论及下一条唯一调整项。未发布时保持空白，不得填写预测值冒充结果。
