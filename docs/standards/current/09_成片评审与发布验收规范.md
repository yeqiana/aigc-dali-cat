# 《啾啾和大力》成片评审与发布验收规范 V1.1｜分级版

> 适用：全部单集  
> 原则：角色、比例、安全和技术交付为共同 P0；故事复杂度按 A-Lite / B / C 分别验收。

---

## 1. 结论

只允许：

1. 正式发布通过；
2. 修改后发布；
3. 不通过。

角色身份、比例、安全、禁重复和水印任一 P0 失败，直接不通过。

---

## 2. 评审流程

1. 文件检查；
2. 正常带声音观看；
3. 静音观看；
4. 逐段暂停检查连续性；
5. 手机缩略图检查；
6. 按模式 P0；
7. 记录数据和成本。

---

## 3. 共同 P0

### 角色

- [ ] 啾啾身份、毛色、脸型和粉色背心连续；
- [ ] 大力身份、奶茶色毛色和脸型连续；
- [ ] 大力只有一顶蓝帽；
- [ ] 大力无衣服、长尾巴和老鼠化；
- [ ] 大力为啾啾肩高 35%—45%；
- [ ] 两者处于同一透视地面；
- [ ] 无额外肢体和畸形爪子。

### 巨物和安全

- [ ] 巨物为最大主体；
- [ ] 巨物参与动作；
- [ ] 巨物与背景分离；
- [ ] 相邻镜头尺寸和状态连续；
- [ ] 无真实撞伤、压伤、夹伤、困住和危险感；
- [ ] 不以疼痛、哭泣和恐惧为笑点。

### 画面和文件

- [ ] 无水印；
- [ ] 无品牌、快递单、随机文字和完整人类；
- [ ] 9:16；
- [ ] 1080 × 1920；
- [ ] H.264 High Profile；
- [ ] 恒定 25 或 30 fps；
- [ ] 视频码率 8—12 Mbps；
- [ ] AAC 48 kHz 立体声；
- [ ] 音频 192 kbps；
- [ ] 真峰值不高于 -1 dBFS。

---

## 4. A-Lite P0

- [ ] 已通过禁重复检查；
- [ ] 第一帧已经发生异常；
- [ ] 1 秒内识别巨物和方向；
- [ ] 2 秒内看懂大力目标或啾啾行动；
- [ ] 全片只有一个核心动作；
- [ ] 动态具有预备、主动作和结果；
- [ ] 无声能理解开始和结果；
- [ ] 有一个清楚结果、轻反转或循环；
- [ ] 不要求两次翻转；
- [ ] 不因缺少 4—6 个片段而判失败；
- [ ] 有巨物音、角色动作音、结果音和短静默；
- [ ] 无凑时长空镜；
- [ ] 尝试次数不超过 4 次或已执行降级。

---

## 5. B 模式 P0

- [ ] 已通过 A-Lite 升级门槛；
- [ ] 当前集没有重演 A-Lite 原测试；
- [ ] 单集只承担一个五集故事节点；
- [ ] 第一帧是新状态，不是重复旧开场；
- [ ] 2.5 秒内建立本集目标；
- [ ] 啾啾偏离与大力计划相关；
- [ ] 无声能理解本集因果；
- [ ] 1—2 段动态均有预备、主动作和结果；
- [ ] 至少有一个有效结果、翻转或循环；
- [ ] 与上一集尾帧连续；
- [ ] 第五集有关系或情绪回报；
- [ ] 声音有事件、主动作、静默、结果和接点。

B 不因只有一个强翻转而失败。

---

## 6. C 模式 P0

- [ ] 已通过免费数据和预算门槛；
- [ ] 精品化提升清楚；
- [ ] 第一帧 0.6 秒内形成强异常；
- [ ] 2 秒内建立目标和关系；
- [ ] 完整动作链无声可懂；
- [ ] 每段动态有预备、主动作和结果；
- [ ] 高潮和最终回报清楚；
- [ ] 相邻镜头角色、巨物、光线和空间连续；
- [ ] 完整声音正文已填写；
- [ ] 高潮前有 0.15—0.25 秒静默；
- [ ] 关键因果音未被配乐遮挡；
- [ ] 未超预算和尝试上限。

C 推荐两次递进结果，但若“一个翻转 + 强情绪回报”更有效，不机械增加第二次事故。

---

## 7. 手机缩略图检查

- [ ] 第一眼看见巨物；
- [ ] 第一眼看见异常方向；
- [ ] 角色轮廓分离；
- [ ] 粉色背心可见；
- [ ] 蓝帽可见且只有一顶；
- [ ] 表情可读；
- [ ] 画面不过暗；
- [ ] 背景不杂乱。

---

## 8. 修改后发布边界

可以后期修复：

- 反应停留略长；
- 某一音效不清楚；
- 循环硬切；
- 标题时机；
- 轻微空镜；
- 不影响身份的单帧小瑕疵。

必须重制：

- 角色身份或比例错误；
- 帽子复制、消失或成为剧情道具；
- 无声看不懂；
- 动作因果不存在；
- 真实危险；
- 与已发布内容高相似；
- 水印无法去除；
- 文件参数严重不达标；
- 需要大量字幕解释。
