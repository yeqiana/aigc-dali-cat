# 《啾啾和大力》最终执行规范 V2.1.1｜实战复盘 + 尺寸比例锁定版

> 用途：后续用于《啾啾和大力》AI 图片、AI 视频、分镜脚本、封面、标题、评论区互动的统一规范。

## 1. 系列名

# 《啾啾和大力》

## 系列一句话定位

> 穿粉色绒绒背心的小母猫啾啾，和戴蓝色棒球帽的小仓鼠大力，在巨大化的日常世界里，因为普通物品发生各种错位小故事。

## 定位修正说明

原先的“认真闯祸、努力解决、温暖收尾”不作为每条视频同时出现的固定结构。  
后续每一条只选择其中一种情绪方向，三者是 **或的关系**，不是并列全塞。

### 单集结尾方向三选一

| 类型 | 含义 | 适合场景 |
|---|---|---|
| 闯祸型 | 结尾停在轻微翻车、搞笑定格 | 西瓜滚走、大力帽子歪、啾啾一脸无辜 |
| 解决型 | 它们用笨办法完成了小任务 | 把纸条送到门口、把奶茶吸管搭成桥 |
| 温暖型 | 不强调解决问题，强调陪伴和治愈 | 两只靠在一起休息、躲雨、看夕阳 |

### 当前执行原则

> 每条视频只选一个主情绪：要么闯祸，要么解决，要么温暖。  
> 不要一条里同时做到“闯祸 + 解决 + 温暖”，否则 10-15 秒视频会太满。

## 2. 视觉风格最终定稿

## 风格关键词

- 2D动画渲染
- 3D体积感
- 真实毛发细节
- 软萌治愈短剧风
- 温暖光影
- 干净背景
- 小动物视角
- 元素错位
- 轻喜剧

## 风格解释

不是纯照片写实，也不是平面Q版插画。  
画面应该像一部高质量二维动画短片的角色和场景，但角色有明显的三维体积、柔和光影、真实毛发、材质细节。

## 必须保持

| 项目 | 规则 |
|---|---|
| 角色质感 | 毛发柔软真实，有体积，有光影 |
| 场景质感 | 干净、温暖、生活化 |
| 色彩 | 柔和、低饱和、奶油感、治愈感 |
| 物品 | 真实日常物品，但在小动物视角下巨大 |
| 镜头 | 适合竖屏短视频，主体清晰，冲突一眼可见 |

## 禁止风格

- 纯照片写实宠物风
- 低龄幼教Q版风
- 扁平平涂卡通风
- 黑暗恐怖风
- 脏乱街头写实风
- 赛博机甲风
- 夸张鬼畜表情包风
- 外卖制服打工风

## 5. 啾啾和大力的身高体重与大小比例锁定

> 本节为生产视觉比例规则，优先级高于所有“明显更小”“小巧”等模糊描述。  
> 身高体重是用于 AI 生图、生视频、关键帧和母版控制的虚拟设定值，不按真实宠物饲养标准执行。

### 5.1 角色基准尺寸

| 角色 | 视觉定位 | 身高 / 肩高 | 体重 | 体长参考 | 说明 |
|---|---|---:|---:|---:|---|
| 啾啾 | 主视觉小猫 | 四足站立肩高约 **24cm**；坐姿到耳尖约 **32cm** | 约 **2.2kg** | 鼻尖到尾根约 **35cm** | 白色银渐层小母猫，粉色绒绒小背心，画面里的中等体量角色 |
| 大力 | 掌心小仓鼠 | 站立到头顶约 **9cm**；含帽顶最高不超过 **10.5cm** | 约 **120g** | 鼻尖到尾根约 **8cm** | 奶茶色男仓鼠，只戴一顶低饱和牛仔蓝小棒球帽，画面里的最小角色 |

### 5.2 画面比例硬规则

| 项目 | 规则 |
|---|---|
| 啾啾视觉基准 | 啾啾四足站立肩高 = **1.0J** |
| 大力推荐高度 | 大力站立到头顶，不含帽，约为啾啾肩高的 **35%-45%** |
| 大力最佳值 | 大力默认按啾啾肩高的 **40%** 生成 |
| 大力上限 | 大力含帽顶也不得超过啾啾肩高的 **50%** |
| 视觉体积 | 大力整体视觉体积约为啾啾的 **1/5 到 1/4** |
| 同框层级 | **巨大物品 > 啾啾 > 大力** |
| 透视条件 | 只有在大力明显更靠近镜头时，才允许视觉上略微变大；同一地面、同一景深时必须严格按比例 |

### 5.3 一票否决

出现以下情况，当前图或视频片段直接重做：

- 大力高度接近或超过啾啾肩高的一半。
- 大力看起来像大型仓鼠玩偶，而不是掌心小仓鼠。
- 大力和啾啾接近同体量。
- 大力因为透视不清而显得比啾啾更抢主视觉。
- 大力长尾巴、尖嘴、瘦长，变成老鼠。
- 大力帽子复制成两顶，或帽子飞离头部。

### 5.4 生图 / 生视频固定写法

```text
啾啾是一只小体型白色银渐层小母猫，四足站立肩高约24cm，坐姿到耳尖约32cm，体重约2.2kg，穿粉色绒绒小背心。

大力是一只掌心大小的奶茶色男仓鼠，站立到头顶约9cm，体重约120g，只戴一顶低饱和牛仔蓝小棒球帽。大力明显小于啾啾，默认只有啾啾肩高的40%左右，含帽顶也不能超过啾啾肩高的一半。画面比例必须始终是：巨大物品 > 啾啾 > 大力。
```

### 5.5 负面提示词补充

```text
不要让大力变成大型仓鼠，不要让大力接近啾啾一半以上的高度，不要让大力和啾啾接近同体量，不要让大力比啾啾更抢主视觉，不要比例混乱，不要把大力画成老鼠，不要长尾巴，不要尖嘴，不要瘦长，不要两顶帽子。
```

## 6. 角色关系

## 关系定位

啾啾和大力不是敌对关系，也不是猫抓鼠关系。  
它们是：

> 互坑互救的迷你搭档。

## 分工关系

| 角色 | 分工 |
|---|---|
| 啾啾 | 行动派，负责先靠近、先尝试、推动事件发生 |
| 大力 | 计划派，负责出主意、瞎指挥、制造反差 |
| 两者共同点 | 都很认真，但经常误判巨大物品的难度 |

## 互动模式

### 模式一：啾啾先冲，大力在旁边指挥

啾啾直接靠近巨大物品，大力站在旁边认真指挥，结果不一定靠谱。

### 模式二：大力先吹牛，啾啾负责观察

大力表现得很自信，结果下一秒被物品带走、压住、吓到、帽子歪掉。

### 模式三：两只都误判

它们都以为这是一个小问题，结果普通物品在小动物视角下变成巨大挑战。

## 7. 世界观设定

## 世界观一句话

> 这是一个小动物视角下的巨大日常世界，所有普通人类物品在啾啾和大力眼里都像一场冒险。

## 世界规则

1. 不做魔法世界，场景来自现实生活。
2. 日常物品保持真实，但比例对小动物来说很巨大。
3. 不出现完整人类主角。
4. 不强调外卖员、职业、工装、打工制服。
5. 冲突是轻喜剧，不做危险惊悚。
6. 每集只讲一个错位笑点。
7. 结尾可以是闯祸、解决或温暖三选一。

## 人类出现规则

### 允许出现

- 一只手递出物品
- 门缝
- 桌上的纸条
- 地上的物品
- 手机提示
- 门口、柜台、货架、玄关
- 鞋子或影子，但不作为主角

### 不建议出现

- 完整人脸
- 完整人体
- 人类对话戏
- 人类追赶小动物
- 人类训斥小动物
- 人类强烈冲突

## 8. 小任务触发方式

## 触发方式定稿

不设定“每日任务”。  
后续统一叫：

> 小任务触发。

小任务可以是很小、很生活化、很普通的事情，但在啾啾和大力视角里变得很夸张。

## 小任务触发来源

| 触发方式 | 示例 |
|---|---|
| 物品自然出现 | 一个巨大西瓜滚到面前 |
| 桌上纸条 | 纸条写着“把它挪开” |
| 门口物品 | 玄关放着一个大快递盒 |
| 手机提示 | 屏幕亮起，但不强调外卖订单 |
| 大力主动提议 | 大力指着巨大物品，表现得很自信 |
| 啾啾误会 | 啾啾把猫粮袋当自己的粮仓 |
| 环境变化 | 下雨了，一把伞倒在地上 |

## 小任务原则

- 小任务要一眼看懂。
- 不需要复杂背景。
- 不要变成完整剧情长片。
- 不要强制每天发生。
- 不要每条都解释来龙去脉。
- 直接给物品和冲突。

## 9. 元素错位设计

## 核心公式

> 小动物 + 巨大日常物品 + 小任务 / 误会 + 轻反差 = 短视频笑点

## 巨大物品错位库

| 物品 | 小动物视角 |
|---|---|
| 西瓜 | 绿色小山 / 会滚动的大球 |
| 奶茶 | 甜味深井 / 小池塘 |
| 披萨 | 圆形广场 |
| 钥匙 | 沉重铁锚 |
| 雨伞 | 移动帐篷 |
| 快递盒 | 小房子 |
| 手机 | 巨型地图屏幕 |
| 吸管 | 救援杆 / 长梯子 |
| 鸡腿 | 香味大山 |
| 猫粮袋 | 巨大粮仓 |
| 毛线球 | 滚动障碍球 |
| 纸巾 | 巨大白色被子 |
| 勺子 | 金属滑梯 |
| 拖鞋 | 洞穴 / 小船 |
| 苹果 | 红色小山丘 |

## 错位设计原则

1. 物品必须一眼看懂。
2. 比例夸张，但不能恐怖。
3. 小动物动作要自然。
4. 解决方式可以笨拙，但不能过度复杂。
5. 每条视频只选一个主物品。
6. 不要把多个巨大物品堆在一条里。

## 10. 当前阶段：巨大物品系列生产流程

> 当前只做“巨大物品系列”。  
> 其他流程保留位置，暂不执行。

## 巨大物品系列公式

> 一个巨大物品 + 啾啾和大力的反应 + 一次尝试 + 一个翻车/反差/收尾。

## 推荐视频长度

前期建议：

- 5 秒单镜头测试
- 10-15 秒轻剧情
- 3 个 5 秒镜头拼接

暂不推荐一上来做 30 秒完整视频。

## 标准 15 秒结构

| 时间 | 内容 |
|---|---|
| 0-3 秒 | 强钩子，直接展示巨大物品或离谱结果 |
| 3-7 秒 | 啾啾和大力开始尝试 |
| 7-11 秒 | 出现错位、翻车、表情反应 |
| 11-15 秒 | 闯祸型 / 解决型 / 温暖型三选一收尾 |

## 5 秒单镜头结构

| 时间 | 内容 |
|---|---|
| 0-1 秒 | 巨大物品直接出现 |
| 1-3 秒 | 啾啾或大力动作尝试 |
| 3-5 秒 | 表情反应 / 小翻车 / 动作定格 |

## 12. 开头钩子规则

## 总原则

每条开头 2-3 秒必须直接爆冲突，不做铺垫。  
不要介绍角色，不要说“今天啾啾和大力……”。  
开头要像一个结果已经发生的事故现场。

## 钩子公式

> 巨大物品 + 离谱结果 + 角色反应

或者：

> 反常识判断 + 下一秒翻车

## 更强钩子模板

### 西瓜类

- **西瓜还没送，大力先被推走了。**
- **这个西瓜一滚，大力直接没影了。**
- **啾啾只是碰了一下，西瓜开始追它们。**
- **大力说它能搬，下一秒它被西瓜搬走了。**
- **这不是西瓜，这是会滚的绿色小山。**

### 奶茶类

- **大力只是想闻一口，差点掉进奶茶里。**
- **这不是奶茶，是大力的游泳池。**
- **啾啾低头看了一眼，大力已经在杯边打滑了。**
- **吸管一倒，直接变成救援梯。**

### 披萨类

- **披萨盒一打开，大力迷路了。**
- **这不是披萨，这是大力的广场。**
- **啾啾在找大力，大力在披萨上找出口。**
- **一块披萨，对大力来说是半个操场。**

### 钥匙类

- **一把钥匙，把大力压沉默了。**
- **大力说问题不大，结果钥匙没动，它先趴了。**
- **这不是钥匙，这是仓鼠版铁锚。**

### 雨伞类

- **伞一打开，啾啾以为房子到了。**
- **下雨了，它们直接住进伞里。**
- **这把伞不是用来撑的，是用来躲猫和仓鼠的。**

### 快递盒类

- **盒子刚打开，啾啾已经住进去了。**
- **这不是快递盒，这是啾啾的新家。**
- **大力还在研究，啾啾已经搬进去了。**

## 不推荐钩子

- 大家好，这是啾啾和大力
- 今天它们又出来玩了
- 你们喜欢小动物吗
- 这是一个治愈故事
- 今天发生了一件有趣的事

## 14. 声音系统

## 当前执行方案

> 不说话，用动作和表情表达。  
> 最多保留动物原本的叫声。  
> 旁白系统保留，但暂不实现。

## 允许声音

| 声音 | 用法 |
|---|---|
| 啾啾轻轻喵 | 表达疑惑、惊讶、撒娇 |
| 大力轻微吱吱声 | 表达慌张、得意、委屈 |
| 物品音效 | 西瓜滚动、奶茶晃动、盒子打开、雨伞弹开 |
| 环境音 | 轻微风声、雨声、室内环境音 |

## 暂不使用

- 角色开口说人话
- 长旁白
- 对口型台词
- 复杂对白气泡
- 大段字幕解释

## 28. AI 视频生成实战复盘规范｜巨大西瓜第 1 条

> 来源：基于「巨大西瓜」这一条的 3 个 5 秒视频实测结果补充。
> 目的：降低后续生成废片率，提高角色稳定、动作可控和剪辑可用性。

### 28.1 三镜头拆分验证有效

本次「巨大西瓜」采用 3 个 5 秒镜头生成，再剪辑成 10-15 秒短视频。实测证明该流程有效。

| 镜头 | 核心作用 | 实测结论 |
|---|---|---|
| 镜头1 | 巨物出现 + 双角色反应 | 巨大西瓜压迫感明确，啾啾和大力的比例关系清楚 |
| 镜头2 | 尝试行动 + 喜剧铺垫 | 啾啾推西瓜、大力在旁边参与，动作关系清楚 |
| 镜头3 | 翻车收尾 + 表情定格 | 大力被带动、啾啾发愣的喜剧效果成立 |

后续继续采用：

```text
镜头1：巨大物品出现 / 角色反应
镜头2：角色尝试 / 努力但效果有限
镜头3：轻微翻车 / 表情定格收尾
```

不要一次生成完整 15 秒视频。

---

### 28.2 每个镜头必须有单一动作目标

每个 5 秒镜头只做一件事。不要在一个镜头里同时塞入多个动作阶段。

| 镜头类型 | 只做这件事 | 不要加入 |
|---|---|---|
| 巨物开场镜头 | 巨大物品出现，角色愣住 | 不要开始推、跑、摔倒 |
| 尝试行动镜头 | 角色努力尝试，但效果有限 | 不要提前翻车 |
| 翻车收尾镜头 | 出现轻微意外，停在表情反应 | 不要继续发展新剧情 |

巨大西瓜案例中：

```text
镜头1：西瓜出现，两只愣住。
镜头2：两只推西瓜，西瓜几乎不动。
镜头3：西瓜轻轻动，大力先翻车。
```

---

### 28.3 关键帧要一帧一图，不使用故事板图

正式图生视频时，只投喂当前镜头的干净关键帧图。

关键帧要求：

- 9:16 竖屏；
- 不要镜头编号；
- 不要底部说明文字；
- 不要故事板边框；
- 不要多格分镜；
- 只保留当前镜头画面；
- 啾啾、大力、巨大物品都要清楚可见。

三格故事板只用于内部确认节奏，不直接喂给视频生成软件。

---

### 28.4 首尾帧使用规则

如果视频软件支持首尾帧，可以使用，但尾帧必须是当前镜头的结束状态，不能是下一个镜头的开始状态。

正确示例：

| 镜头 | 首帧 | 尾帧 |
|---|---|---|
| 镜头1 | 西瓜刚出现 | 西瓜停住，两只愣住 |
| 镜头2 | 开始推西瓜 | 更努力推，但西瓜仍几乎没动 |
| 镜头3 | 西瓜轻轻动，大力开始踉跄 | 大力停住，帽子歪，啾啾发愣 |

错误做法：

```text
把镜头2首帧当镜头1尾帧。
把镜头3首帧当镜头2尾帧。
```

这样会导致剧情提前发生，镜头节奏变乱。

---

### 28.5 道具状态必须唯一

小道具在视频模型里最容易复制，尤其是帽子、纸条、钥匙、吸管等可移动物件。

大力的帽子必须明确数量和状态：

```text
大力在整个镜头里始终只有一顶低饱和牛仔蓝小棒球帽。
```

当前阶段推荐：

```text
帽子始终戴在大力头上，只是在动作过程中明显歪向一侧。
帽子不要脱离头部，不要飞起来，不要掉下来，不要复制出第二顶帽子。
```

暂不推荐：

```text
帽子飞起来。
帽子掉下来。
帽子被甩飞。
```

原因：飞帽动作容易生成「头上一顶 + 空中一顶」的双帽问题。

---

### 28.6 提示词不要写“或”

视频提示词里避免使用：

- 或；
- 可以；
- 也可以；
- 飞起或歪掉；
- 掉下来或留在头上；
- 被带跑或摔倒。

错误示例：

```text
帽子被甩飞或明显歪掉。
```

正确示例：

```text
帽子始终戴在大力头上，只是明显歪向一侧。
```

原则：

> 一个镜头里，一个角色、一个道具、一个动作，只允许一种最终状态。

---

### 28.7 巨大物品运动要轻，不要变成危险事故

巨大物品可以有压迫感，但不能出现危险感。

巨大西瓜类镜头推荐写法：

```text
西瓜只缓慢滚动一小段，动作幅度很小，有重量感，不快速撞击角色，不压住角色。
```

不要写：

```text
西瓜快速滚过来。
西瓜追着它们跑。
西瓜撞飞大力。
```

除非后续明确要做更强动作片段，否则当前阶段保持轻喜剧。

---

### 28.8 镜头内要保留运动空间

巨大物品不能把画面完全占满，否则角色没有动作空间。

推荐构图：

| 元素 | 画面占比建议 |
|---|---|
| 巨大物品 | 40%-60% |
| 啾啾 + 大力 | 25%-40% |
| 环境留白 | 10%-20% |

如果巨大物品超过 70%，角色动作容易被挤压，后续剪辑也缺少空间感。

---

### 28.9 镜头衔接不追求绝对无缝

3 个 5 秒 AI 片段之间，可以接受轻微位置变化。

必须保证的是：

1. 啾啾稳定；
2. 大力稳定；
3. 巨大物品一眼看懂；
4. 动作逻辑顺；
5. 观众能看懂「出现 → 尝试 → 翻车」。

不用为了完全无缝连续，牺牲角色稳定性。

剪辑时可以用轻微音效、直接切、动作点切换来掩盖位置变化。

---

### 28.10 结尾必须停在反应上

短视频结尾不要停在动作进行中，要停在反应表情上。

推荐结尾：

- 大力帽子歪在头上，愣住；
- 啾啾抬着前爪，一脸无辜；
- 西瓜停住，两只一起看向它；
- 大力摔坐在旁边，但没有受伤；
- 啾啾和大力同时沉默。

不推荐结尾：

- 角色还在跑；
- 西瓜还在滚；
- 镜头还在晃；
- 动作没有停；
- 新剧情刚开始。

原则：

> 结尾停在表情，不停在动作。

---

### 28.11 音效后期添加，不依赖视频生成

当前阶段不说话、不做旁白。

视频软件生成时可以保留简单环境音，但关键音效建议后期在剪映添加。

巨大西瓜案例推荐音效：

| 位置 | 音效 |
|---|---|
| 镜头1 | 低沉轻微滚动声 + 惊讶小音效 |
| 镜头2 | 爪子摩擦声 + 小动物用力声 |
| 镜头3 | 轻微滚动声 + 帽子歪掉音效 + 大力慌张吱吱声 |

音效要轻，不要盖过画面。

---

### 28.12 单镜头负面提示词要按剧情补充

通用负面提示词不够。每个镜头都要加当前镜头专属限制。

镜头1补充：

```text
不要让啾啾和大力开始推西瓜，不要提前进入镜头2。
```

镜头2补充：

```text
不要让西瓜明显滚走，不要让大力被带跑，不要帽子飞起来，不要提前进入镜头3。
```

镜头3补充：

```text
不要复制帽子，不要出现两顶帽子，不要一顶帽子在头上同时另一顶帽子在空中，不要西瓜压住角色，不要大力被甩得太远。
```

### 28.13 生成前检查表

每个 5 秒镜头生成前，先检查：

| 检查项 | 是否通过 |
|---|---|
| 是否只投喂当前镜头内容？ | 是 / 否 |
| 关键帧是否是干净单帧图？ | 是 / 否 |
| 是否没有镜头编号、底部说明和边框？ | 是 / 否 |
| 当前镜头是否只做一件事？ | 是 / 否 |

| 大力是否始终只有一顶帽子？ | 是 / 否 |

### 28.14 生成后验收标准

每个 5 秒镜头生成完后检查：

| 检查项 | 合格标准 |
|---|---|
| 角色稳定 | 啾啾、大力没有变形，没有多出角色 |
| 服装稳定 | 啾啾粉色背心稳定，大力蓝帽稳定 |
| 道具唯一 | 没有两顶帽子，没有多余西瓜 |
| 动作单一 | 只完成当前镜头动作 |
| 剧情节奏 | 没有提前进入下一镜头 |
| 氛围 | 可爱、轻松、不危险 |
| 画面 | 没有乱码、说明文字、品牌 |
| 比例 | 巨大物品 > 啾啾 > 大力 |
| 可剪辑性 | 结尾能自然接下一个镜头 |

### 28.15 本次巨大西瓜复盘结论

本条可用经验：

1. 3 个 5 秒拆镜头是当前最稳流程。
2. 巨大物品开场有效，比例反差明显。
3. 大力的帽子是高风险道具，优先采用“歪帽不飞帽”。
4. 镜头2要专门限制“西瓜不明显滚走”，避免提前进入翻车。
5. 镜头3要专门限制“帽子不复制、角色不受伤、物品不快速滚动”。
6. 结尾应停在表情反应上，方便观众理解笑点。
7. 不追求三个片段绝对无缝，优先保证角色和动作逻辑。
8. 音效建议后期添加，不依赖视频模型一次性生成。

最终规则：

> 宁可动作简单，也要角色稳定；宁可镜头短，也要笑点清楚；宁可帽子歪，不要帽子飞。
