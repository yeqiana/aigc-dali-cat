## 0. 当前阶段最高优先级

后续每条内容生产前，先看这 8 条。

1. **宁可动作简单，也要角色稳定。**
2. **宁可镜头短，也要笑点清楚。**
3. **宁可帽子歪，不要帽子飞。**
4. **每个 5 秒镜头只做一件事。**
5. **巨大物品要有压迫感，但不能变成危险事故。**
6. **镜头替代对白，观众不看字幕也要看懂。**
7. **高潮前必须有停顿，结尾必须停在反应。**
8. **声音只强化动作和情绪，不解释剧情。**

### 一票否决项

如果出现以下任一问题，当前视频片段直接重做：

- 啾啾换品种、换衣服、换成其他猫。
- 大力变成老鼠、长尾巴、变大、穿衣服。
- 大力出现两顶帽子。
- 巨大物品压住角色、撞飞角色、造成危险感。
- 出现完整人类主角或清晰人脸。
- 出现乱码文字、品牌 Logo、镜头编号、说明文字。
- 0-3 秒看不懂巨大物品和冲突。

## 2.2 风格解释

画面不是纯照片写实，也不是平面 Q 版插画。  
应该像一部高质量二维动画短片，但角色和物体有明显三维体积、柔和光影、真实毛发和材质细节。

## 2.3 必须保持

| 角色质感 | 毛发柔软真实，有体积，有光影 |

## 3.3 身高体重与大小比例锁定

> 本节为生产视觉比例规则，优先级高于所有“明显更小”“小巧”等模糊描述。  
> 身高体重是用于 AI 生图、生视频、关键帧和母版控制的虚拟设定值，不按真实宠物饲养标准执行。

### 3.3.1 角色基准尺寸

| 角色 | 视觉定位 | 身高 / 肩高 | 体重 | 体长参考 | 说明 |
|---|---|---:|---:|---:|---|
| 啾啾 | 主视觉小猫 | 四足站立肩高约 **24cm**；坐姿到耳尖约 **32cm** | 约 **2.2kg** | 鼻尖到尾根约 **35cm** | 白色银渐层小母猫，粉色绒绒小背心，画面里的中等体量角色 |
| 大力 | 掌心小仓鼠 | 站立到头顶约 **9cm**；含帽顶最高不超过 **10.5cm** | 约 **120g** | 鼻尖到尾根约 **8cm** | 奶茶色男仓鼠，只戴一顶低饱和牛仔蓝小棒球帽，画面里的最小角色 |

### 3.3.2 画面比例硬规则

| 项目 | 规则 |
|---|---|
| 啾啾视觉基准 | 啾啾四足站立肩高 = **1.0J** |
| 大力推荐高度 | 大力站立到头顶，不含帽，约为啾啾肩高的 **35%-45%** |
| 大力最佳值 | 大力默认按啾啾肩高的 **40%** 生成 |
| 大力上限 | 大力含帽顶也不得超过啾啾肩高的 **50%** |
| 视觉体积 | 大力整体视觉体积约为啾啾的 **1/5 到 1/4** |
| 同框层级 | **巨大物品 > 啾啾 > 大力** |
| 透视条件 | 只有在大力明显更靠近镜头时，才允许视觉上略微变大；同一地面、同一景深时必须严格按比例 |

### 3.3.3 一票否决

出现以下情况，当前图或视频片段直接重做：

- 大力高度接近或超过啾啾肩高的一半。
- 大力看起来像大型仓鼠玩偶，而不是掌心小仓鼠。
- 大力和啾啾接近同体量。
- 大力因为透视不清而显得比啾啾更抢主视觉。
- 大力长尾巴、尖嘴、瘦长，变成老鼠。
- 大力帽子复制成两顶，或帽子飞离头部。

### 3.3.4 生图 / 生视频固定写法

```text
啾啾是一只小体型白色银渐层小母猫，四足站立肩高约24cm，坐姿到耳尖约32cm，体重约2.2kg，穿粉色绒绒小背心。

大力是一只掌心大小的奶茶色男仓鼠，站立到头顶约9cm，体重约120g，只戴一顶低饱和牛仔蓝小棒球帽。大力明显小于啾啾，默认只有啾啾肩高的40%左右，含帽顶也不能超过啾啾肩高的一半。画面比例必须始终是：巨大物品 > 啾啾 > 大力。
```

### 3.3.5 负面提示词补充

```text
不要让大力变成大型仓鼠，不要让大力接近啾啾一半以上的高度，不要让大力和啾啾接近同体量，不要让大力比啾啾更抢主视觉，不要比例混乱，不要把大力画成老鼠，不要长尾巴，不要尖嘴，不要瘦长，不要两顶帽子。
```

## 3.4 角色关系

啾啾和大力不是敌对关系，也不是猫抓鼠关系。  
它们是：

> **互坑互救的迷你搭档。**

| 角色 | 分工 |
|---|---|
| 啾啾 | 行动派，负责先靠近、先尝试、推动事件发生 |
| 大力 | 计划派，负责出主意、瞎指挥、制造反差 |
| 两者共同点 | 都很认真，但经常误判巨大物品的难度 |

常用互动模式：

1. 啾啾先冲，大力在旁边指挥。
2. 大力先吹牛，啾啾负责观察。
3. 两只都误判普通物品的难度。

## 4. 世界观与小任务

## 4.1 世界观一句话

> 这是一个小动物视角下的巨大日常世界，所有普通人类物品在啾啾和大力眼里都像一场冒险。

## 4.2 世界规则

1. 不做魔法世界，场景来自现实生活。
2. 日常物品保持真实，但比例对小动物来说很巨大。
3. 不出现完整人类主角。
4. 不强调外卖员、职业、工装、打工制服。
5. 冲突是轻喜剧，不做危险惊悚。
6. 每集只讲一个错位笑点。
7. 结尾可以是闯祸、解决或温暖三选一。

## 4.3 人类出现规则

### 允许出现

- 一只手递出物品。
- 门缝。
- 桌上的纸条。
- 地上的物品。
- 手机提示。
- 门口、柜台、货架、玄关。
- 鞋子或影子，但不作为主角。

### 不建议出现

- 完整人脸。
- 完整人体。
- 人类对话戏。
- 人类追赶小动物。
- 人类训斥小动物。
- 人类强烈冲突。

---

## 4.4 小任务触发方式

不设定“每日任务”。后续统一叫：

> **小任务触发。**

小任务可以是很小、很生活化、很普通的事情，但在啾啾和大力视角里会变得很夸张。

| 触发方式 | 示例 |
|---|---|
| 物品自然出现 | 一个巨大西瓜滚到面前 |
| 桌上纸条 | 纸条写着“把它挪开” |
| 门口物品 | 玄关放着一个大快递盒 |
| 手机提示 | 屏幕亮起，但不强调外卖订单 |
| 大力主动提议 | 大力指着巨大物品，表现得很自信 |
| 啾啾误会 | 啾啾把猫粮袋当自己的粮仓 |
| 环境变化 | 下雨了，一把伞倒在地上 |

小任务原则：

- 小任务要一眼看懂。
- 不需要复杂背景。
- 不要变成完整剧情长片。
- 不要强制每天发生。
- 不要每条都解释来龙去脉。
- 直接给物品和冲突。

## 5. 元素错位设计

## 5.1 核心公式

> **小动物 + 巨大日常物品 + 小任务 / 误会 + 轻反差 = 短视频笑点**

## 5.2 巨大物品错位库

| 物品 | 小动物视角 |
|---|---|
| 西瓜 | 绿色小山 / 会滚动的大球 |
| 奶茶 | 甜味深井 / 小池塘 |
| 披萨 | 圆形广场 |
| 钥匙 | 沉重铁锚 |
| 雨伞 | 移动帐篷 |
| 快递盒 | 小房子 |
| 手机 | 巨型地图屏幕 |
| 吸管 | 救援杆 / 长梯子 |
| 鸡腿 | 香味大山 |
| 猫粮袋 | 巨大粮仓 |
| 毛线球 | 滚动障碍球 |
| 纸巾 | 巨大白色被子 |
| 勺子 | 金属滑梯 |
| 拖鞋 | 洞穴 / 小船 |
| 苹果 | 红色小山丘 |

## 5.3 错位设计原则

1. 物品必须一眼看懂。
2. 比例夸张，但不能恐怖。
3. 小动物动作要自然。
4. 解决方式可以笨拙，但不能过度复杂。
5. 每条视频只选一个主物品。
6. 不要把多个巨大物品堆在一条里。

## 6. 当前阶段：巨大物品系列生产流程

当前只做“巨大物品系列”。其他流程保留位置，暂不执行。

## 6.1 巨大物品系列公式

> **一个巨大物品 + 啾啾和大力的反应 + 一次尝试 + 一个翻车 / 反差 / 收尾。**

## 6.2 推荐视频长度

前期建议：

- 5 秒单镜头测试。
- 10-15 秒轻剧情。
- 3 个 5 秒镜头拼接。

暂不推荐一上来做 30 秒完整视频。

## 6.3 标准 15 秒结构

| 时间 | 内容 |
|---|---|
| 0-3 秒 | 强钩子，直接展示巨大物品或离谱结果 |
| 3-7 秒 | 啾啾和大力开始尝试 |
| 7-11 秒 | 出现错位、翻车、表情反应 |
| 11-15 秒 | 闯祸型 / 解决型 / 温暖型三选一收尾 |

## 6.4 5 秒单镜头结构

| 时间 | 内容 |
|---|---|
| 0-1 秒 | 巨大物品直接出现 |
| 1-3 秒 | 啾啾或大力动作尝试 |
| 3-5 秒 | 表情反应 / 小翻车 / 动作定格 |

## 6.5 单条内容最小模板

```text
物品：
钩子：
结尾类型：闯祸型 / 解决型 / 温暖型
镜头1：巨大物品出现
镜头2：啾啾和大力尝试
镜头3：翻车 / 解决 / 温暖定格
标题：
评论区：
```

## 6.6 15 秒节奏曲线

当前阶段推荐 10-15 秒轻剧情。15 秒不是平均分配动作，而是按喜剧节奏组织：

| 时间 | 节奏 | 内容 | 目的 |
|---|---|---|---|
| 0-2 秒 | 快 | 巨大物品出现，钩子成立 | 让观众立刻知道“发生了离谱的事” |
| 2-4 秒 | 慢 | 啾啾和大力看清物品，表情反应 | 让观众消化比例反差 |
| 4-8 秒 | 中 | 啾啾和大力开始尝试行动 | 建立“它们很认真”的喜剧基础 |
| 8-10 秒 | 停顿 | 翻车前的蓄力，角色短暂停住 | 制造“要出事了”的预期 |
| 10-12 秒 | 快 | 小翻车发生 | 释放笑点 |
| 12-15 秒 | 慢 | 表情定格，笑点收住 | 给观众理解和回味的时间 |

### 节奏原则

1. 高潮前必须有 **0.5-1 秒停顿**。
2. 翻车动作不要从第一秒就开始，要先让观众看懂“谁在做什么”。
3. 结尾不要停在动作进行中，要停在表情反应上。
4. 如果只有 10 秒，也保留“看清 → 尝试 → 停顿 → 翻车 → 反应”的节奏。

## 6.7 5 秒镜头三拍规则

每个 5 秒镜头内部也要有三拍，不要把 5 秒全部用来做动作。

| 时间 | 功能 | 说明 |
|---|---|---|
| 0-1 秒 | 信息进入 | 巨大物品、动作起点、角色视线 |
| 1-3 秒 | 动作发生 | 推、拉、闻、躲、滑、滚、探头 |
| 3-5 秒 | 反应定格 | 震惊、无辜、嘴硬、狼狈、沉默 |

### 巨大西瓜示例

```text
镜头1：
0-1秒：西瓜缓慢滚入。
1-3秒：啾啾和大力抬头看。
3-5秒：两只愣住。

镜头2：
0-1秒：啾啾贴爪准备推。
1-3秒：啾啾和大力一起用力。
3-5秒：西瓜几乎不动，两只还在努力。

镜头3：
0-1秒：西瓜轻轻动。
1-3秒：大力踉跄，帽子歪。
3-5秒：大力停住，啾啾发愣。
```

### 三拍原则

> 每个镜头最后至少留 **1 秒表情反应**。  
> 动作是笑点的原因，反应才是笑点的落地。

## 7. 单镜头投喂标准包

这是每次给 AI 视频生成软件的最小输入。生成每个 5 秒镜头时，只给当前镜头内容。

## 7.1 必须投喂

1. 当前镜头干净关键帧图。
2. 当前镜头正向提示词。
3. 当前镜头负面提示词。
4. 时长：5 秒。
5. 比例：9:16。
6. 运动强度：低 / 中。
7. 镜头运动：低机位、轻微推进或轻微跟随。

## 7.2 禁止投喂

- 整份 MD。
- 其他镜头内容。
- 标题。
- 评论区文案。
- 故事板说明文字。
- 镜头编号。
- 底部说明。
- 三格故事板图。

原则：

> 视频软件只需要知道“这个 5 秒里发生什么”，不要让它理解整集。

## 8. 关键帧和首尾帧规则

## 8.1 关键帧必须一帧一图

正式图生视频时，关键帧图必须是一帧一张，不使用三格故事板图。

关键帧要求：

- 9:16 竖屏。
- 不要镜头编号。
- 不要底部说明文字。
- 不要故事板边框。
- 不要多格分镜。
- 只保留当前镜头画面。
- 啾啾、大力、巨大物品都要清楚可见。

三格故事板只用于内部确认节奏，不直接喂给视频生成软件。

## 8.2 首尾帧规则

如果视频软件支持首尾帧，可以使用，但尾帧必须是当前镜头的结束状态，不能是下一个镜头的开始状态。

| 镜头 | 首帧 | 尾帧 |
|---|---|---|
| 镜头1 | 西瓜刚出现 | 西瓜停住，两只愣住 |
| 镜头2 | 开始推西瓜 | 更努力推，但西瓜仍几乎没动 |
| 镜头3 | 西瓜轻轻动，大力开始踉跄 | 大力停住，帽子歪，啾啾发愣 |

错误做法：

```text
把镜头2首帧当镜头1尾帧。
把镜头3首帧当镜头2尾帧。
```

这样会导致剧情提前发生，镜头节奏变乱。

## 9. 提示词规则

## 9.1 每个镜头只做一件事

每个 5 秒镜头只允许有一个核心动作。

| 镜头类型 | 只做这件事 | 不要加入 |
|---|---|---|
| 巨物开场镜头 | 巨大物品出现，角色愣住 | 不要开始推、跑、摔倒 |
| 尝试行动镜头 | 角色努力尝试，但效果有限 | 不要提前翻车 |
| 翻车收尾镜头 | 出现轻微意外，停在表情反应 | 不要继续发展新剧情 |

巨大西瓜案例：

```text
镜头1：西瓜出现，两只愣住。
镜头2：两只推西瓜，西瓜几乎不动。
镜头3：西瓜轻轻动，大力先翻车。
```

## 9.2 提示词不要写“或”

视频提示词里避免使用：

- 或；
- 可以；
- 也可以；
- 飞起或歪掉；
- 掉下来或留在头上；
- 被带跑或摔倒。

错误示例：

```text
帽子被甩飞或明显歪掉。
```

正确示例：

```text
帽子始终戴在大力头上，只是明显歪向一侧。
```

原则：

> 一个镜头里，一个角色、一个道具、一个动作，只允许一种最终状态。

## 9.3 道具状态必须唯一

小道具在视频模型里最容易复制，尤其是帽子、纸条、钥匙、吸管等可移动物件。

大力的帽子必须明确数量和状态：

```text
大力在整个镜头里始终只有一顶低饱和牛仔蓝小棒球帽。
```

当前阶段推荐：

```text
帽子始终戴在大力头上，只是在动作过程中明显歪向一侧。帽子不要脱离头部，不要飞起来，不要掉下来，不要复制出第二顶帽子。
```

暂不推荐：

```text
帽子飞起来。
帽子掉下来。
帽子被甩飞。
```

原因：飞帽动作容易生成“头上一顶 + 空中一顶”的双帽问题。

## 10. 运动强度和巨大物品规则

## 10.1 巨大物品运动要轻

巨大物品可以有压迫感，但不能出现危险感。

巨大西瓜类镜头推荐写法：

```text
西瓜只缓慢滚动一小段，动作幅度很小，有重量感，不快速撞击角色，不压住角色。
```

不要写：

```text
西瓜快速滚过来。
西瓜追着它们跑。
西瓜撞飞大力。
```

当前阶段保持轻喜剧，不做危险动作。

## 10.2 镜头内要保留运动空间

巨大物品不能把画面完全占满，否则角色没有动作空间。

| 元素 | 画面占比建议 |
|---|---|
| 巨大物品 | 40%-60% |
| 啾啾 + 大力 | 25%-40% |
| 环境留白 | 10%-20% |

如果巨大物品超过 70%，角色动作容易被挤压，后续剪辑也缺少空间感。

## 10.3 推荐运动强度

| 镜头类型 | 运动强度 |
|---|---|
| 巨物出现 | 低 |
| 角色反应 | 低 |
| 推、拉、闻、爬 | 低到中 |
| 小翻车 | 中 |
| 追逐、滚动 | 中，不建议高 |

禁止：

- 快速镜头旋转。
- 大幅推拉。
- 剧烈抖动。
- 角色飞出画面。
- 物品高速撞击角色。

## 15. 常见生成问题排查表

| 问题 | 可能原因 | 修正方式 |
|---|---|---|
| 大力出现两顶帽子 | 提示词写了“飞起或歪掉” | 只保留“帽子歪在头上”，负面词加“不要两顶帽子” |
| 镜头2提前翻车 | 没有限制不要进入镜头3 | 加“西瓜不要明显滚走，不要大力被带跑” |
| 西瓜压住角色 | 巨物运动过强 | 改成“缓慢滚动一小段，不撞击角色” |
| 大力变成老鼠 | 仓鼠特征不够强 | 强调“圆滚滚男仓鼠、无长尾、不是老鼠” |
| 大力变大 | 比例锁定不够 | 加“大力站立到头顶约9cm，默认只有啾啾肩高40%左右，含帽顶也不能超过啾啾肩高50%” |
| 啾啾换衣服 | 服装锁定不够 | 重复“粉色绒绒小背心，白色包边，不换衣服” |
| 画面出现字幕乱码 | 参考图或提示词有文字 | 用干净关键帧，负面词加“不要文字、不要乱码” |
| 三个镜头不连续 | AI 分镜天然有偏差 | 接受轻微偏差，用剪辑和音效衔接 |
| 动作太大变危险 | 运动强度过高 | 降低运动强度，改成轻微移动、轻喜剧 |
| 巨大物品看不懂 | 构图太近或物品太局部 | 拉远一点，保留物品完整轮廓和纹理 |
| 角色被物品遮挡 | 巨物占比太大 | 巨物控制在画面 40%-60%，保留角色运动空间 |
| 角色表情不明显 | 镜头太远 | 结尾加表情定格，保证啾啾和大力脸部可见 |

## 17. 生成后验收标准

每个 5 秒镜头生成完后检查：

| 检查项 | 合格标准 |
|---|---|
| 角色稳定 | 啾啾、大力没有变形，没有多出角色 |
| 服装稳定 | 啾啾粉色背心稳定，大力蓝帽稳定 |
| 道具唯一 | 没有两顶帽子，没有多余西瓜 |
| 动作单一 | 只完成当前镜头动作 |
| 剧情节奏 | 没有提前进入下一镜头 |
| 氛围 | 可爱、轻松、不危险 |
| 画面 | 没有乱码、说明文字、品牌 |
| 比例 | 巨大物品 > 啾啾 > 大力 |
| 可剪辑性 | 结尾能自然接下一个镜头 |

## 18. 剪辑衔接规则

三个 5 秒 AI 片段之间，可以接受轻微位置变化。

必须保证的是：

1. 啾啾稳定。
2. 大力稳定。
3. 巨大物品一眼看懂。
4. 动作逻辑顺。
5. 观众能看懂“出现 → 尝试 → 翻车”。

不用为了完全无缝连续，牺牲角色稳定性。

推荐衔接方式：

- 直接切。
- 轻微“咚”音效。
- 西瓜滚动声延续。
- 小动物惊讶音效。

不推荐：

- 炫酷闪白。
- 强节奏卡点。
- 大幅镜头旋转。
- 多字幕解释。

---

## 19. 结尾规则

短视频结尾不要停在动作进行中，要停在反应表情上。

推荐结尾：

- 大力帽子歪在头上，愣住。
- 啾啾抬着前爪，一脸无辜。
- 西瓜停住，两只一起看向它。
- 大力摔坐在旁边，但没有受伤。
- 啾啾和大力同时沉默。

不推荐结尾：

- 角色还在跑。
- 西瓜还在滚。
- 镜头还在晃。
- 动作没有停。
- 新剧情刚开始。

原则：

> **结尾停在表情，不停在动作。**

## 20. 声音设计系统

## 20.1 当前执行方案

> 当前阶段不做角色人话配音，不做长旁白，不做对口型。  
> 声音只服务三个目标：**强化动作、强化情绪、强化节奏点**。

《啾啾和大力》优先使用零对白视觉叙事。观众不听旁白，也应该能通过画面看懂剧情。声音只做辅助，不替画面解释。

## 20.2 三层声音结构

| 层级 | 内容 | 作用 |
|---|---|---|
| 环境底音 | 室内环境、轻微风声、地毯摩擦、远处生活环境音 | 保持画面自然，不让视频太空 |
| 动作音效 | 西瓜滚动、奶茶晃动、纸盒打开、帽子歪、爪子摩擦 | 强化动作节点，让笑点更清楚 |
| 情绪音效 | 啾啾轻喵、大力吱吱、惊讶短音、轻微滑稽音 | 替代表情对白，强化角色反应 |

## 20.3 暂不使用

- 角色开口说人话。
- 长旁白。
- 对口型台词。
- 复杂对白气泡。
- 大段字幕解释。
- 过强 BGM 盖住动作音效。

## 20.4 BGM 规则

当前阶段可以使用统一轻喜剧 BGM，但不能喧宾夺主。

### 推荐 BGM

- 轻快。
- 短循环。
- 木琴、拨弦、轻打击、软萌小鼓点。
- 温暖、调皮、轻喜剧。
- 音量低于关键动作音效。

### 不推荐 BGM

- 热血燃曲。
- 土味 DJ。
- 悲伤钢琴。
- 恐怖悬疑。
- 强鼓点卡点。
- 过度煽情。

## 20.5 声音使用原则

1. BGM 负责系列氛围。
2. 物品音效负责当前集记忆点。
3. 情绪音效负责替代对白。
4. 关键笑点前后音效要短，不要拖太长。
5. 如果音效会干扰观众理解画面，宁可不用。

## 20.6 巨大西瓜案例声音设计

| 位置 | 声音设计 |
|---|---|
| 镜头1 | 低沉轻微滚动声 + 小惊讶音 + 啾啾轻轻“喵” |
| 镜头2 | 爪子摩擦声 + 西瓜轻微受力声 + 大力短促吱吱声 |
| 镜头3 | 轻微滚动声 + 帽子歪掉小音效 + 大力慌张吱吱声 |

音效要轻，不要让视频变成吵闹搞笑音效合集。

## 21. 镜头语言

## 21.1 最高原则：展示，不解释

《啾啾和大力》当前阶段不依赖对白、旁白和字幕解释。镜头必须替代对白。

观众不看文字，也应该能通过画面理解：

1. 谁看见了什么；
2. 谁想做什么；
3. 哪里出了问题；
4. 最后谁懵了。

如果一个镜头必须靠文字解释才能看懂，说明镜头设计还不够清楚。

## 21.2 固定镜头库

| 镜头 | 用法 |
|---|---|
| 巨物开场镜头 | 巨大物品直接占据画面，让观众立刻看懂冲突 |
| 双角色反应镜头 | 啾啾和大力同时愣住、看向物品、互相看一眼 |
| 视线匹配镜头 | 角色看向某处 → 目标物出现 → 角色反应 |
| 尝试行动镜头 | 推、拉、爬、闻、躲、探头、抱住 |
| 翻车镜头 | 大力被带走、帽子歪、啾啾僵住、物品滚动 |
| 表情定格镜头 | 大眼睛、无辜、震惊、嘴硬、委屈 |
| 停顿镜头 | 高潮前短暂停住，制造“要出事了”的预期 |
| 温暖收尾镜头 | 两只靠在一起、躲在物品旁、安静休息 |

## 21.3 视觉叙事五法则

### 法则一：看 → 目标 → 反应

不用台词解释“它们看到了什么”，用镜头讲清楚。

```text
角色看向某处 → 巨大物品 / 问题出现 → 角色表情反应
```

巨大西瓜示例：

```text
啾啾抬头看 → 巨大西瓜滚入 → 啾啾和大力同时愣住
```

### 法则二：反应镜头至少保留 1 秒

没有对白时，表情就是对白。每个镜头最后至少留 1 秒给角色反应。

可用反应：

- 啾啾眼睛瞪圆；
- 啾啾一只前爪悬在空中；
- 大力张嘴愣住；
- 大力帽子歪在头上；
- 两只同时沉默看向巨大物品。

### 法则三：高潮前必须有停顿

翻车前不要直接冲出去。先停一下，让观众预判“要出事了”。

```text
大力自信停住 → 西瓜轻轻动 → 大力踉跄
```

停顿时长建议：**0.5-1 秒**。

### 法则四：固定机位优先，动作靠角色完成

非动作镜头优先固定机位，减少视觉负担。动作感主要来自角色和物品，不靠镜头乱晃。

推荐：

- 低机位固定镜头；
- 轻微推进；
- 轻微跟随；
- 主体始终清楚。

不推荐：

- 快速旋转；
- 大幅推拉；
- 强烈晃动；
- 频繁变焦；
- 角色高速冲向镜头。

### 法则五：轻量 Z 轴，保留空间层次

允许使用前景、中景、背景制造空间感，但不要复杂穿梭。

推荐层次：

| 层次 | 内容 |
|---|---|
| 前景 / 左前景 | 巨大物品，比如西瓜、奶茶、披萨 |
| 中景 | 啾啾和大力的主要动作区 |
| 背景 | 干净玄关、桌面、地毯、家具轮廓 |

原则：

> 巨大物品有压迫感，但角色必须有运动空间。

## 21.4 单条视频推荐镜头组合

### 10-15 秒

> 巨物开场 → 视线匹配 / 双角色反应 → 尝试行动 → 停顿 → 翻车 / 收尾 → 表情定格

### 5 秒单镜头

> 信息进入 → 动作发生 → 反应定格

## 21.5 镜头设计自查

每个镜头设计完后，问自己四个问题：

1. 不看提示词，观众能看懂角色在看什么吗？
2. 不加旁白，观众能看懂角色想做什么吗？
3. 翻车前有没有停顿？
4. 结尾有没有停在表情上？

如果答案是否定，先改镜头，不要急着生成视频。

## 23. 开头钩子规则

## 23.1 总原则

每条开头 2-3 秒必须直接爆冲突，不做铺垫。  
不要介绍角色，不要说“今天啾啾和大力……”。  
开头要像一个结果已经发生的事故现场。

## 23.2 钩子公式

> 巨大物品 + 离谱结果 + 角色反应

或者：

> 反常识判断 + 下一秒翻车

## 23.3 更强钩子模板

### 西瓜类

- 西瓜还没送，大力先被推走了。
- 这个西瓜一滚，大力直接没影了。
- 啾啾只是碰了一下，西瓜开始追它们。
- 大力说它能搬，下一秒它被西瓜搬走了。
- 这不是西瓜，这是会滚的绿色小山。

### 奶茶类

- 大力只是想闻一口，差点掉进奶茶里。
- 这不是奶茶，是大力的游泳池。
- 啾啾低头看了一眼，大力已经在杯边打滑了。
- 吸管一倒，直接变成救援梯。

### 披萨类

- 披萨盒一打开，大力迷路了。
- 这不是披萨，这是大力的广场。
- 啾啾在找大力，大力在披萨上找出口。
- 一块披萨，对大力来说是半个操场。

### 钥匙类

- 一把钥匙，把大力压沉默了。
- 大力说问题不大，结果钥匙没动，它先趴了。
- 这不是钥匙，这是仓鼠版铁锚。

### 雨伞类

- 伞一打开，啾啾以为房子到了。
- 下雨了，它们直接住进伞里。
- 这把伞不是用来撑的，是用来躲猫和仓鼠的。

### 快递盒类

- 盒子刚打开，啾啾已经住进去了。
- 这不是快递盒，这是啾啾的新家。
- 大力还在研究，啾啾已经搬进去了。

## 23.4 不推荐钩子

- 大家好，这是啾啾和大力。
- 今天它们又出来玩了。
- 你们喜欢小动物吗。
- 这是一个治愈故事。
- 今天发生了一件有趣的事。

## 24.1 封面规范：保留设定，暂时不强制执行

## 24.2 标题规范

标题先不做系列化。  
发布标题暂时不强制带“第几集”。

标题原则：

- 不强制写《啾啾和大力》第 X 集。
- 不强制带连续编号。

- 直接说冲突和反差。

推荐标题公式：

> 巨大物品 + 离谱结果

或者：

> 角色行为 + 意外反转

标题示例：

系列信息可以放在简介或合集里：

> 《啾啾和大力》小动物错位短剧

## 24.3 评论区互动

互动继续放在评论区、标题、简介，不放进视频本体。

- 视频本体不为互动多做镜头。

## 25. 版权边界

总原则：

> **不抄袭。**

可以借鉴动物喜剧的通用表达，但不能复刻任何已有动画的具体角色、桥段、分镜、台词或造型。

## 25.1 可以借鉴

- 小动物搭档喜剧。
- 体型反差。
- 道具错位。
- 追逐节奏。
- 误会反转。
- 一大一小的动作喜剧。

## 25.2 禁止借鉴

- 具体角色外形。
- 具体经典桥段。
- 具体分镜构图。
- 具体台词。
- 具体动作设计。
- 具体颜色组合和角色关系。

## 25.3 原创核心

> 啾啾和大力是猫和仓鼠搭档，不是猫抓老鼠。  
> 它们是朋友，是一起面对巨大日常物品的小动物组合。

## 28. 巨大西瓜实战复盘

## 28.1 有效经验

1. 3 个 5 秒拆镜头有效。
2. 巨物开场有效，比例反差明显。
3. 尝试行动镜头能建立“它们很认真”的喜剧基础。
4. 结尾停在表情反应上，更容易让观众理解笑点。
5. 三个片段不需要绝对无缝，轻微位置变化可以通过剪辑接受。

## 28.2 出现问题

1. 大力帽子容易复制。
2. 西瓜运动过强容易变成危险感。
3. 镜头2如果不限制，容易提前进入镜头3。
4. 镜头3如果动作过大，容易把轻喜剧变成事故。
5. 参考图如果带文字、边框、镜头编号，容易影响视频生成。

## 28.3 规则更新

1. 帽子只歪不飞。
2. 巨物只轻微滚动。
3. 每个镜头只做一个动作。
4. 尾帧只能是当前镜头结束状态。
5. 镜头专属负面词必须补充。
6. 不追求绝对无缝，优先角色稳定。
7. 后续所有镜头补充“看 → 目标 → 反应”和“高潮前停顿”。
8. 声音只做动作和情绪辅助，不做剧情解释。

## 28.4 下次复用结构

```text
镜头1：巨物出现 + 角色愣住
镜头2：角色尝试 + 物品几乎不动
镜头3：轻微翻车 + 表情定格
```

## 30. 最终定稿摘要

## 系列名

> 《啾啾和大力》

## 视觉风格

> 2D动画渲染 + 3D体积感 + 真实毛发细节 + 软萌治愈短剧风。

## 角色锁定

> 啾啾：粉色绒绒背心白色银渐层小母猫。  
> 大力：蓝色棒球帽奶茶色小男仓鼠。

## 世界观

> 小动物视角下的巨大日常世界。

## 当前生产流程

> 巨大物品系列。

## 内容公式

> 巨大日常物品 + 啾啾和大力 + 小任务 / 误会 + 一个错位笑点。

## 视频结构

> 0-3 秒爆炸钩子，3-7 秒尝试，7-11 秒错位翻车，11-15 秒按单条选择闯祸型 / 解决型 / 温暖型之一收尾。

## 当前声音方案

> 不说话，不做旁白，用动作和表情表达；用三层声音设计辅助：环境底音、动作音效、情绪音效。

## 暂时保留但不实现

- 结尾固定梗。
- 旁白系统。
- 封面规范。
- 系列化标题。
- 其他生产流程。

## 最终执行原则

> **宁可动作简单，也要角色稳定；宁可镜头短，也要笑点清楚；宁可帽子歪，不要帽子飞；镜头替代对白，高潮前有停顿，结尾停在反应。**
