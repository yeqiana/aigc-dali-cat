# 《啾啾和大力》A-Lite 低成本生产规范 V1.0

> 生效日期：2026-07-11  
> 适用范围：路线图第 1—30 集，以及任何尚未通过数据验证的新题材。  
> 定位：在不降低角色身份、比例和安全 P0 的前提下，降低动态片段数、关键帧数和声音制作成本。  
> 与仓库 `main` 关系：本文件是 A-Lite 单集的模式补充；角色、比例、安全、无品牌和无文字规则继续服从当前主规范。

---

# 1. 模式选择

立项时必须先选择：

```text
A-Lite / B / C
```

## 使用 A-Lite 的条件

- 题材尚未验证；
- 当前主要使用免费额度；
- 单集只有一个核心动作；
- 6—8 秒可以讲清楚；
- 不需要多场景、多角色或复杂救援；
- 主要目的是测试点击、完播、动作结构或角色关系。

## 不得使用 A-Lite 的情况

- 需要跨场景连续叙事；
- 需要两次清晰翻转才能成立；
- 需要 3 个以上角色；
- 动作中包含高速遮挡、复杂缠绕或危险碰撞；
- 是已经验证的数据胜出题材，准备做精品重制。

# 2. A-Lite 与当前强节奏规范的差异

| 项目 | A-Lite | B / 当前常规 | C 精品 |
|---|---|---|---|
| 推荐时长 | 6—8 秒 | 8—12 秒 | 12—20 秒 |
| 生成动态 | 1 段 | 1—2 段 | 2—4 段 |
| 动态时长 | 3—5 秒 | 每段 3—5 秒 | 视故事拆分 |
| 关键帧 | 1—2 张 | 2—3 张 | 3—6 张 |
| 场景 | 1 个 | 1 个为主 | 可扩展 |
| 核心动作 | 1 个 | 1 个节点 | 完整动作链 |
| 翻转 | 1 个轻反转或循环即可 | 建议 1—2 个 | 完整两次翻转 |
| 生成片段数 | 不强制 4—6 个 | 2—4 个 | 4—6 个 |
| 声音 | 简化时间轴 | 标准时间轴 | 完整动作同步管弦乐 |
| 单镜头尝试 | 最多 4 次 | 最多 6 次 | 按预算审批 |

> A-Lite 不要求为了满足格式而拆出 4—6 个生成片段，也不要求每条都完成两次视觉翻转。

# 3. 不可降低的 P0

无论成本多低，以下规则不能放宽：

## 啾啾

- 白色银渐层小母猫；
- 大头、短圆四肢、四足小猫体态；
- 始终穿粉色绒绒背心和白色包边；
- 不换品种、不换衣服、不增加项圈或复杂配饰；
- 表情依靠眼睛、嘴型、耳朵和身体重心，不能像真实宠物静态照片。

## 大力

- 奶茶色小男仓鼠；
- 圆头、短手短脚；
- 约为啾啾肩高 35%—45%；
- 含帽最高不超过啾啾肩高 50%；
- 只戴一顶低饱和牛仔蓝棒球帽；
- 不穿衣服、不长尾巴、不变成老鼠、不复制帽子。

## 共同

- 巨物 > 啾啾 > 大力；
- 同一透视地面；
- 无完整人类、品牌、快递单、随机文字和水印；
- 无真实伤害、压伤、夹伤、危险撞击和恐怖感；
- 帽子不作为交通工具或整集主道具；
- 选题必须通过禁重复清单。

任意一项失败，成片不发布。

# 4. A-Lite 标准结构

## 4.1 6—8 秒结构

| 时间 | 功能 | 必须表达 | 推荐实现 |
|---:|---|---|---|
| 0.0—0.8 秒 | 异常事件 | 巨物已经在运动或变形 | 关键帧快速推近或动态段直接开始 |
| 0.8—2.0 秒 | 目标 / 反应 | 大力想处理，啾啾准备行动 | 动态段前半段 |
| 2.0—5.5 秒 | 唯一核心动作 | 只做一项主要运动 | 图生视频核心段 |
| 5.5—7.5 秒 | 结果 | 轻反转、表情或循环接点 | 动态尾帧或静态关键帧加剪辑 |

## 4.2 第一帧最低要求

第一帧至少包含三项：

- [ ] 巨物或异常结构；
- [ ] 明确运动方向或形变趋势；
- [ ] 至少一个角色身体反应；
- [ ] 主体与背景色彩分离；
- [ ] 不看字幕也会产生问题。

禁止开头：

- 两只角色站着看；
- 稳定摆放的普通道具；
- 慢慢展示房间；
- 先出现标题再发生事件；
- 只靠旁白解释。

# 6. 关键帧规则

## 数量

- 默认 1 张首帧 + 1 张结果帧；
- 若图生视频只需要单图，可只保留 1 张强首帧；
- 不为展示完整剧情额外生成九格图；
- 九格只作为内部草图，不作为 A-Lite 必需交付物。

## 单张要求

- 一图只有一个动作重点；
- 角色、巨物和运动方向在手机缩略图可读；
- 大力比例处于 35%—45%；
- 啾啾背心和大力唯一蓝帽可见；
- 预留动作方向空间；
- 不出现标题、镜头编号、对白和多格排版。

## 推荐提示词骨架

```text
9:16 竖屏 A-Lite 首帧关键帧；
啾啾与大力的身份、服装和标准比例；
一件巨大日常物品正在发生明确异常；
一个唯一动作重点；
角色通过眼睛、嘴型、耳朵、四肢、身体重心或帽子角度表达反应；
干净居家地毯或玄关；
低机位，中景或大全景；
主体与背景具有明确色相或明暗分离；
高品质 3D 动画电影角色，块状柔软绒毛，非真实宠物摄影；
不要文字、品牌、完整人类、额外动物、危险伤害、服装变化、比例漂移或帽子复制。
```

# 7. 动态段规则

## 一个动态段只允许

```text
预备 → 主动作 → 结果
```

示例：

```text
纸袋先轻微颤动
→ 快速鼓起并向前滑动
→ 泄气后停在两只角色面前
```

不要在同一段同时要求：

- 巨物进入；
- 啾啾追赶；
- 大力搭机关；
- 第一次失败；
- 第二次救援；
- 新角色出现。

## 提示词原则

- 动作使用具体动词，不写“发生有趣的事情”；
- 明确起点、方向、速度变化和尾帧；
- 双角色同时高速运动时，优先降级为一只角色运动、另一只反应；
- 不要求大幅旋转角色身体；
- 避免角色长时间被巨物遮挡；
- 结尾必须恢复角色正常比例。

# 8. 简化声音规范

A-Lite 不强制为每条写完整六阶段管弦乐正文，但必须保留最小声音因果：

## 最小配置

- 1 条固定可复用的无对白轻喜剧 BGM；
- 1 个巨物声音；
- 1 个角色动作声音；
- 1 个结果或翻转声音；
- 1 个 0.15—0.25 秒短静默；
- 结尾不使用沉重终止音。

## 简化时间轴

```text
0.0—0.8 秒：巨物事件音 + BGM 立即进入
0.8—2.0 秒：轻快蓄力音型
2.0—5.5 秒：核心动作音，BGM 随速度略加速
翻转前：0.15—0.25 秒静默
5.5—7.5 秒：轻巧结果音 + 极短木管尾音
```

禁止：

- 对白、旁白、歌词；
- “meow”等拟声文字；
- 全程满音轨；
- 伤害、疼痛和沉重碰撞声音；
- BGM 遮住关键因果音。

# 9. 四次尝试止损

## 第 1 次失败

只修正最明显的一个问题：动作方向、角色比例或道具结构。

## 第 2 次失败

减少一个同时运动的角色，保留唯一核心动作。

## 第 3 次失败

降低速度、缩小位移，固定镜头。

## 第 4 次失败

停止继续消耗额度，选择以下降级方式：

1. 双角色运动改为单角色运动；
2. 角色运动改为道具运动；
3. 高速动作改为缓慢动作；
4. 用两张静态关键帧表现开始和结果；
5. 用剪辑推镜、摇镜、速度变化和震动补足；
6. 用音效表现无法稳定生成的中间过程；
7. 更换为下一候选选题。

# 10. A-Lite P0 发布验收

- [ ] 已通过禁重复检查；
- [ ] 第一帧已经发生事件；
- [ ] 1 秒内可识别巨物和运动方向；
- [ ] 2 秒内能看懂大力目标或啾啾行动；
- [ ] 全片只有一个核心动作；
- [ ] 无声观看能理解开始和结果；
- [ ] 啾啾身份、脸型和粉色背心连续；
- [ ] 大力身份、比例和唯一蓝帽连续；
- [ ] 大力没有长尾巴、衣服或体型放大；
- [ ] 巨物与背景具有色彩或明暗分离；
- [ ] 角色不像真实宠物摄影；
- [ ] 没有真实伤害或危险感；
- [ ] 没有文字、品牌、完整人类和额外角色；
- [ ] 有关键动作音和短静默；
- [ ] 成片没有为凑时长保留的空镜；
- [ ] 尝试次数和返工成本已记录。

任意一项失败，不进入正式发布。

# 11. 数据升级门槛

A-Lite 题材满足以下任意两项，可升级为 B：

- 完播率高于近 10 条平均值；
- 平均播放时长高于近 10 条平均值；
- 重播率明显提升；
- 评论主动出现角色名；
- 主页访问或涨粉高于平均；
- 4 次以内可以稳定生成；
- 同一动作结构连续两条表现较好。

升级为 C 还需满足：

- 已有至少 2 条免费版本验证；
- 角色一致性可控；
- 题材具备明确的情绪高潮或视觉回报；
- 付费重制后可以明显提升，而不是只增加时长。
