# 《啾啾和大力》1.0 → main 强节奏动画版迁移说明

## 1. 分支策略

- `1.0`：冻结旧版完整项目，不再接收新版生产规则。
- `main`：当前强节奏动画生产体系，后续新单集只在此分支维护。

旧版快照提交：`a6f0f33`。

## 2. 主要迁移

| 1.0 | main 当前版 |
|---|---|
| 多份 V2.1.1、V2.2.1、V2.3.1 执行规范 | 单一 `01_最终执行规范.md` |
| 单集填空表 V1.1 | `02_单集生产表.md` |
| 母版清单 V1.1 | `03_母版清单.md` |
| 第一批 6 张母版提示词 V1.0 | `04_全量母版生成提示词.md` |
| 尺寸比例更新说明 V1.0 | `05_尺寸与比例规范.md` |
| 3 个 5 秒镜头 | 4–6 个 1.3–2.5 秒片段 |
| 10–15 秒默认成片 | 8–10 秒优先 |
| 低运动、固定机位 | 开头允许中高运动，镜头服务故事状态 |
| 真实毛发细节优先 | 动画电影头身、块状绒毛、夸张表演优先 |
| 单次轻微翻转 | 两次视觉翻转或循环结尾 |

## 3. 删除与保留策略

main 不再保留旧生产文档作为当前入口，避免复制错版本。以下内容仍保留：

- 研究资料；
- 已交付视频和关键帧；
- 成片复盘；
- 巨大快递盒重制执行表；
- 旧位图母版，作为身份和构图参考。

完整旧文档通过 `1.0` 分支访问。

## 4. 位图母版迁移

现有位图母版生成于 1.0 写实倾向规则下。main 使用方式调整为：

- 可参考角色颜色、服装、体型关系和空间构图；
- 不复制真实宠物毛发、写实动物解剖和静态表情；
- 新图必须叠加 `04_全量母版生成提示词.md` 的动画化风格底座；
- 新母版通过 P0 验收后替换同编号当前资产；
- 未通过验收的图片不能作为视频强参考图。

## 5. 单集迁移

旧单集需要重制时：

1. 保留旧成片和关键帧作为复盘证据。
2. 使用 `02_单集生产表.md` 新建重制执行表。
3. 第一帧改为异常事件，不复用纯站立观看开头。
4. 将 5 秒单状态片段拆成 1.3–2.5 秒因果片段。
5. 增加第二次翻转和循环接点。
6. 按 1080 × 1920、H.264、8–12 Mbps 重新导出。

## 6. 回滚

若 main 新流程出现不可接受问题：

```text
git switch 1.0
```

不在 main 内恢复旧文档副本，以免再次形成多个生效入口。
