# 《啾啾和大力》最终执行规范 V3.0｜强节奏动画锁定版

> 状态：main 分支当前锁定执行规范
> 依据：《巨大快递盒》成片复盘与改进建议 V1.0
> 目标：提高前 3 秒留存、画面信息密度、动画角色表演和结尾复看价值
> 不变项：角色身份、服装、体型比例、安全边界和无品牌要求
> 旧版入口：Git 分支 `1.0`

## 1. 版本定位

本文件是 `main` 分支唯一有效的总执行规范，用于替代旧流程中“3 个 5 秒片段、低运动强度、固定机位、写实毛发优先”的生产方式。

执行优先级：

1. 本文件负责内容公式、视觉风格、节奏、表演、提示词、剪辑、声音和发布验收。
2. `02_单集生产表.md` 负责每条视频的具体填写与执行。
3. `03_母版清单.md` 与 `04_全量母版生成提示词.md` 负责视觉资产生产。
4. `05_尺寸与比例规范.md` 负责角色和巨物的硬比例约束。
5. `07_配乐与音效执行规范.md` 负责动作同步管弦乐、因果音效、静默和循环尾音的硬约束。
6. 单集文件可以补充当前剧情限制，但不能降低本文件的 P0 验收标准。

### 1.1 系列定位

```text
在普通人类居家空间里，一只好奇、先行动的白色小猫和一只认真、爱指挥的小仓鼠，把日常巨物误用成新的游戏、任务和生活空间。
```

### 1.2 内容公式

```text
巨大日常物品 + 明确任务 + 角色目标错位 + 安全动作翻车 + 两次视觉翻转 + 可循环结尾
```

### 1.3 角色关系

- 啾啾和大力是搭档，不是捕食关系。
- 大力负责计划、解释和努力解决问题。
- 啾啾负责观察、快速行动和温和偏离计划。
- 笑点来自目标错位，不来自伤害、羞辱或角色失败惩罚。

### 1.4 世界规则

- 场景以干净的居家空间为主。
- 日常物品相对角色呈现巨大比例。
- 允许局部人类痕迹，如手、鞋、家具和声音；不出现完整人类主角。
- 巨物可以滑动、弹起、晃动、滚动和折叠，但动作必须安全、可爱、无真实伤害。
- 画面不出现品牌、快递单、复杂文字、血腥和恐怖元素。

## 2. 核心改动

| 旧规则 | 新规则 | 原因 |
|---|---|---|
| 3 个 5 秒片段拼成 10–15 秒 | 4–6 个 1.3–2.5 秒片段拼成 8–10 秒 | 避免用停顿填满时长 |
| 每个 5 秒镜头只做一件事 | 每个片段只完成一个因果动作，允许预备、爆发、结果三个状态 | 一件事不等于一个静态状态 |
| 开头运动强度低 | 开头允许中高运动，结尾按需要回落 | 第一帧必须形成事件 |
| 固定低机位、轻微推进 | 根据故事目标选择大全景、中景、近景和短促跟拍 | 景别变化服务信息变化 |
| 不要提前进入下一镜头 | 当前片段必须完成自己的因果动作，尾帧为下一片段提供动作接点 | 避免生成模型不敢推进 |
| 真实毛发细节优先 | 卡通化头身比例、块状柔软绒毛、清晰眉眼和夸张嘴型优先 | 提高动画 IP 感和表情可读性 |
| 最后至少停 1 秒 | 反应只保留到观众读懂，通常 0.3–0.8 秒 | 防止成片拖沓 |
| 持续环境音 | 动作音、停顿、翻转音效形成强弱对比 | 声音必须推动节奏 |

## 3. 成片结构

### 3.1 推荐时长

- 常规轻喜剧：8–10 秒。
- 单一强梗：6–8 秒。
- 有两次翻转：9–12 秒。
- 不再默认要求凑满 15 秒。

### 3.2 推荐片段数量

- 4–6 个生成片段。
- 单片段建议 1.3–2.5 秒。
- 复杂动作拆成多个片段，不要求一个生成片段完成整段故事。
- 每个片段只保留一个主动作或一个主反应。

### 3.3 标准喜剧节奏

| 阶段 | 推荐时间 | 必须回答的问题 |
|---|---:|---|
| 异常事件 | 0–0.6 秒 | 什么东西不正常？ |
| 目标建立 | 0.6–1.8 秒 | 角色想做什么？ |
| 行动冲突 | 1.8–3.5 秒 | 谁的行为偏离了目标？ |
| 停顿蓄力 | 3.5–4.2 秒 | 结果会是什么？ |
| 第一次翻转 | 4.2–6.2 秒 | 预期如何被打破？ |
| 第二次翻转 | 6.2–8.5 秒 | 为什么值得笑和重看？ |
| 循环接点 | 最后 0.5 秒 | 如何自然接回开头？ |

## 4. 信息密度规则

### 4.1 第一帧

第一帧不是场景介绍，必须已经包含以下至少三项：

- 巨物；
- 明确运动方向；
- 角色身体反应；
- 目标或阻碍；
- 反常结果；
- 高识别度色彩对比。

禁止使用以下内容作为唯一开头：

- 两只角色站着看；
- 两只角色坐着发愣；
- 巨物稳定摆放；
- 缓慢推镜展示环境；
- 只靠标题解释发生了什么。

### 4.2 状态变化

每 0.8–1.5 秒至少改变一项：

- 构图；
- 角色姿势；
- 表情；
- 巨物状态；
- 角色关系；
- 声音强弱；
- 观众已知信息。

只有景别变化、没有故事状态变化，不计为有效信息变化。

### 4.3 一个片段的三状态

每个 2–3 秒动作片段建议包含：

```text
预备姿势 → 主动作爆发 → 结果姿势
```

示例：

```text
大力顶住纸箱蓄力 → 双脚快速打滑仍拼命推 → 纸箱不动，大力贴在箱壁喘气
```

这仍然是一个因果动作，不属于过度塞剧情。

## 5. 角色视觉规范

### 5.1 共同风格

统一使用：

```text
高品质3D动画电影角色，卡通化头身比例，柔软块状绒毛，清晰眉眼和夸张嘴型，强轮廓表演，允许安全的挤压拉伸，保留立体电影光影，非真实宠物摄影
```

降低以下关键词权重：

- 逐根真实毛发；
- 真实宠物摄影；
- 写实动物解剖；
- 全程低饱和；
- 全程柔和静态光影。

### 5.2 啾啾角色卡

- 身份：白色银渐层小母猫。
- 体型：四足站立肩高约 24 cm。
- 头身：头部偏大、四肢短圆、脸部轮廓清晰。
- 服装：粉色绒绒小背心，白色包边，始终保持一致。
- 性格：好奇、行动快、容易先冲、闯祸后仍显得无辜。
- 表演重点：大眼、耳朵方向、身体压低与拉长、满足眯眼。
- 禁止变化：不能换品种、换衣服、增加多余配饰或变成真实成年猫比例。

### 5.3 大力角色卡

- 身份：奶茶色小男仓鼠。
- 体型：约为啾啾肩高的 40%，含帽最高不超过啾啾肩高的 50%。
- 头身：圆头、短手短脚、身体小而有弹性。
- 服装：只戴一顶低饱和牛仔蓝小棒球帽，不穿其他衣服。
- 性格：认真、爱指挥、嘴硬、努力解决问题。
- 表演重点：快速小步、双爪指挥、全身僵住、帽子歪斜。
- 禁止变化：不能变大、变成老鼠、长尾巴、复制帽子或更换帽子颜色。

### 5.4 表情验收

每个主要反应至少同时改变以下两项：

- 眼睛或瞳孔；
- 嘴型；
- 耳朵方向；
- 身体重心；
- 四肢姿势；
- 帽子角度。

只改变眼神或轻微张嘴，不算夸张表情。

## 6. 巨物与场景规范

### 6.1 巨物必须可表演

巨物不能只是尺寸很大的静态背景，至少应有一个可参与动作的结构：

- 可弹起的盖子；
- 可滚动的圆形轮廓；
- 可晃动的把手；
- 可拖拽的绳子；
- 可折叠的边缘；
- 可产生遮挡和揭示的开口。

### 6.2 色彩分离

- 巨物与背景不能长期处于相同色相和明度。
- 至少设置一个高识别度辅助色。
- 啾啾粉色背心和大力蓝色帽子必须保持可见。
- 背景服务主体，不追求所有区域都温暖低饱和。
- 第一帧缩小为手机预览尺寸后仍应一眼识别巨物和角色反应。

```text
巨物辅助色：
角色可活动区域：
镜头可用前景：
```

## 7. 故事板规则

### 7.1 一格一拍

故事板的一格只表现以下一种内容：

- 一个动作；
- 一个反应；
- 一个物体状态；
- 一个关系变化；
- 一个视觉记忆点。

一格中禁止同时塞入多个翻转。

### 7.2 九格节奏模板

| 格数 | 功能 |
|---:|---|
| 01 | 巨物或异常事件进入 |
| 02 | 双角色即时反应 |
| 03 | 主角目标建立 |
| 04 | 另一角色偏离目标 |
| 05 | 主动作爆发 |
| 06 | 短暂停顿或遮挡 |
| 07 | 第一次翻转揭示 |
| 08 | 第二次翻转动作 |
| 09 | 结尾表情与循环接点 |

九格用于规划，不代表成片必须出现九次硬切。

关键帧必须做到：

- 主体、动作和情绪各只有一个重点；
- 不依赖上下文也能读懂；
- 不包含镜头编号、解释文字或多格排版；
- 能作为视频片段明确的首帧或尾帧。

## 9. 视频提示词模板

### 9.1 正向结构

```text
生成一个[时长]秒、9:16竖屏的高品质3D动画电影片段。

首帧状态：[角色、巨物、空间位置和情绪]。
0–[时间]秒预备：[唯一预备动作]。
[时间]–[时间]秒主动作：[唯一主动作和物体反馈]。
[时间]–结尾结果：[明确结果姿势和下一片段动作接点]。

镜头：[景别、机位、是否跟随、何时停止]。
表演：[眼睛、嘴型、耳朵、身体重心或帽子至少两项变化]。
色彩：[主体色、辅助色和背景分离]。
连续性：[角色比例、服装和唯一配饰]。
配乐与音效设计：[按 `07_配乐与音效执行规范.md` 填写完整时间轴、动作同步、声音层级、静默点、循环尾音和负面约束]。
```

最终视频提示词中的“配乐与音效设计”是必填正文，不是后期备注。禁止用“轻快音乐”“加入动作音效”或一行“声音参考”代替完整设计。

### 9.2 负面约束分层

第一层只保留身份约束：

```text
不要换角色品种，不要换衣服，不要改变体型比例，不要增加角色，不要复制帽子。
```

第二层保留安全约束：

```text
不要真实伤害，不要血腥，不要危险撞击，不要角色被压伤或卡住。
```

第三层保留画面约束：

```text
不要文字，不要字幕，不要水印，不要品牌logo，不要多格分镜，不要完整人类。
```

避免堆叠以下抑制剧情的负面词：

- 不要进入下一镜头；
- 不要物体明显移动；
- 不要角色被带跑；
- 不要明显翻车；
- 不要新剧情；
- 运动强度只能低。

如果某个行为会破坏当前片段，只描述清楚尾帧状态，不要用大量“不要”封死动作空间。

## 10. 剪辑规则

1. 前 0.6 秒必须已经发生异常事件。
2. 前 3 秒完成异常、目标和冲突三项信息。
3. 每 0.8–1.5 秒至少有一次有效状态变化。
4. 相邻镜头不能只改变景别。
5. 第一次翻转后停顿通常不超过 0.8 秒。
6. 笑点兑现后立即进入第二次翻转或循环接点。
7. 字幕只解释冲突或强化笑点，不重复动物叫声。
8. 不为了时长保留无动作空镜。

---

## 11. 声音规则

声音制作统一执行 `07_配乐与音效执行规范.md`。每条视频至少包含：

- 一条与动作同步的复古无对白动画喜剧管弦乐时间轴；
- 一个明确的巨物声音；
- 一个角色动作声音；
- 一个短静默；
- 一个翻转音效；
- 一个结尾反应声音。

声音结构建议：

```text
强事件音 → 短静默 → 快动作音 → 停顿 → 翻转音效 → 短反应音
```

不让背景音乐持续填满所有节奏点。最终峰值建议保留至少 1 dB 余量。

最终视频提示词必须逐时间段说明：配器与节奏、动作同步点、因果音效、高潮前静默、翻转重音、结尾反应和未解决循环音型。缺少任一项，提示词不得进入视频生成。

## 12. 导出规范

| 项目 | 建议值 |
|---|---|
| 封装 | MP4 |
| 视频编码 | H.264 High Profile |
| 分辨率 | 1080 × 1920 |
| 帧率 | 恒定 25 或 30 fps |
| 视频码率 | 8–12 Mbps |
| 音频 | AAC，48 kHz，立体声 |

| 峰值 | 不高于 -1 dBFS |

发布平台有专用规范时，以平台规范为准。

## 14. P0 验收清单

- [ ] 第一帧已经发生事件，而不是角色站着观看。
- [ ] 0.6 秒内能识别巨物和角色反应。
- [ ] 前 3 秒能看懂目标和冲突。
- [ ] 无声观看仍能理解剧情。
- [ ] 每个片段都有预备、爆发和结果状态。
- [ ] 相邻镜头至少改变一个故事状态。
- [ ] 每个主要反应同时改变至少两项表演元素。
- [ ] 角色具有动画电影感，不像真实宠物摄影。
- [ ] 巨物与背景具有明确色彩或明暗分离。
- [ ] 结尾存在第二次翻转或循环动作。
- [ ] 最终视频提示词已包含完整配乐与音效时间轴，而非简短声音备注。
- [ ] 配乐随动作改变速度和方向，高潮前有可感知静默，结尾使用未解决循环音型。
- [ ] 全片无对白、旁白、歌词和无意义叫声，关键因果音效没有被配乐遮挡。
- [ ] 成片没有无信息停顿。
- [ ] 导出分辨率和码率达到交付要求。

任意一项 P0 未通过，不进入正式发布。

## 15. 数据复盘与版本升级

每条正式发布视频都记录：

- 前 2 秒留存率；
- 5 秒留存率；
- 完播率；
- 重复播放率；
- 点赞、评论和转发率；
- 生成失败率和返工次数。

单集数据结论先回写到该集复盘。只有至少两条视频重复出现同一问题时，才创建新的规范版本；不在当前锁定文件中直接堆叠临时补丁。
